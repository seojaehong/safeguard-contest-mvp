# 49점 목표 로드맵

## 목표
서류심사 상위권 + 발표/시연에서 감점 요소를 최소화해 **49점급 완성도**를 노린다.

## 핵심 원칙
1. **문서 완성도**: 심사표 항목마다 바로 점수를 줄 수 있게 구조화
2. **데모 완성도**: 골든패스 1개는 반드시 끝까지 성공
3. **검증 데이터**: dry-run + live 결과를 누적 기록해 "된다"를 증명
4. **리스크 통제**: 429/timeout이 나와도 데모/설명이 무너지지 않게 fallback 확보

## 49점 기준 체크포인트
### 1. 공공데이터·AI 활용성
- [ ] 어떤 API를 왜 쓰는지 표로 정리
- [ ] 기상청 / KOSHA / 법령의 역할 분담이 선명함
- [ ] AI가 단순 요약이 아니라 산출물 생성에 쓰임

### 2. 실용성
- [ ] 위험성평가 초안 생성
- [ ] TBM 브리핑 생성
- [ ] TBM 일지 초안 생성
- [ ] 외국인/신규 작업자용 쉬운 문장 안내 확장 가능

### 3. 완성도
- [ ] 웹 MVP 동작
- [ ] 대표 시나리오 1개 고정
- [ ] live 일부 실패에도 데모 완주 가능
- [ ] 시연 자료/문서/발표가 서로 같은 메시지를 말함

### 4. 증빙력
- [ ] dry-run 누적 로그
- [ ] live test 누적 로그
- [ ] 실패 유형과 fallback 정책 기록
- [ ] 개선 전/후 비교 가능

## 실증 검증 목표
### 1차 목표
- [ ] 골든패스 dry-run **5회 연속 성공**
- [ ] 골든패스 live **3회 이상 실행**
- [ ] live 결과에서 실패 API와 성공 API 분리 기록

### 2차 목표
- [ ] 다른 입력 시나리오 2개 추가
- [ ] 총 dry-run **10회 이상**
- [ ] 총 live **5회 이상**
- [ ] retry/backoff 도입 후 전/후 비교

## 감점 방지 포인트
- 법적 판단을 단정하지 않는다
- 상용화 완료처럼 과장하지 않는다
- 다국어/TTS는 본체가 아니라 확장 포인트로 유지한다
- 실제 검증된 것과 아직 기획인 것을 분리한다
