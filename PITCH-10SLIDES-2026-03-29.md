# API 활용 대회 발표 초안 — 10장 구성

프로젝트명(가안): **산업안전 위험성평가·TBM 코파일럿**
부제: **공공데이터와 AI로 오늘 작업의 위험을 읽고, 브리핑하고, 기록까지 남기는 현장형 안전 도우미**

---

## 1장. 한 줄 소개
### 제목
산업안전 위험성평가·TBM 코파일럿

### 핵심 메시지
- 기상청·KOSHA·법령 공공데이터를 AI와 결합해
- 오늘 작업의 위험요인을 분석하고
- **위험성평가 초안 / TBM 브리핑 / TBM 일지**까지 생성하는 실무형 서비스

### 발표 멘트 초안
"저희는 공공데이터를 단순 조회에서 끝내지 않고, 산업안전 현장에서 바로 실행 가능한 산출물로 바꾸는 AI 코파일럿을 제안합니다."

---

## 2장. 문제 정의
### 제목
현장에서는 정보가 많아도, 판단은 여전히 어렵습니다

### 핵심 포인트
- 기상특보, 법령, 사고사례, 가이드는 각각 흩어져 있음
- 현장 관리자는 이를 직접 조합해 위험성평가와 TBM을 준비해야 함
- 실제로는 시간이 부족해 형식적 TBM, 반복 문구, 누락된 위험요인이 발생함

### 문제 문장
- "오늘 강풍주의보인데 작업을 멈춰야 하나?"
- "이 상황에서 어떤 법적 근거를 봐야 하나?"
- "TBM은 했는데 실질적 논의가 아니라 체크만 하고 끝난다"

### 발표 멘트 초안
"문제는 데이터 부족이 아니라, 현장에서 바로 판단 가능한 형태로 연결되지 않는다는 점입니다."

---

## 3장. 해결책
### 제목
날씨 → 법령 → 현장조치 → 산출물

### 서비스 흐름
1. 오늘 작업 정보 입력
2. 기상/법령/가이드/사례 데이터 결합
3. AI가 위험요인과 조치 포인트 정리
4. 결과를 현장 산출물로 출력

### 산출물
- 위험성평가 초안
- TBM 브리핑 문안
- TBM 일지 초안
- 실질적 TBM 유도 질문

### 발표 멘트 초안
"저희는 답변만 주는 AI가 아니라, 현장에서 실제로 쓰는 산출물까지 만들어주는 도구를 목표로 합니다."

---

## 4장. 주요 사용자와 사용 장면
### 제목
누가 언제 쓰는가

### 주요 사용자
- 안전관리자
- 관리감독자
- 현장소장 / 반장
- 협력업체 관리자

### 사용 장면
- 아침 작업 전 TBM 준비
- 기상특보 발생 시 작업 지속/중지 판단 보조
- 신규 작업 전 위험성평가 초안 작성
- 교육/점검/브리핑 기록 정리

### 발표 멘트 초안
"특히 시간이 부족한 아침 브리핑과 돌발 기상상황 대응에서 가장 큰 효용이 있습니다."

---

## 5장. 공공데이터 활용 방식
### 제목
공공데이터를 실무 판단 엔진으로 재구성

### 활용 데이터
- **기상청**
  - 단기예보
  - 기상특보
  - 영향예보
- **KOSHA**
  - 안전보건법령 스마트검색
  - 국내재해사례
  - 물질안전보건자료(MSDS)
  - 사고사망 사례 / 안전보건자료 링크
- **법령/판례**
  - Law.go / 국가법령정보

### 활용 방식
- 날씨 신호 감지
- 관련 법령 조문/근거 연결
- KOSHA 가이드 및 사례 보강
- AI가 실무 언어로 재정리

### 발표 멘트 초안
"핵심은 데이터를 많이 보여주는 것이 아니라, 지금 상황과 연결되는 근거만 골라 바로 행동으로 번역하는 것입니다."

---

## 6장. 산출물 예시
### 제목
출력은 보고서가 아니라 현장 실행물입니다

### 예시 1: 위험성평가 초안
- 작업명
- 유해위험요인
- 위험도
- 감소대책
- 관련 법령/가이드

### 예시 2: TBM 브리핑
- 오늘 핵심 위험 3개
- 반드시 지킬 조치
- 날씨 관련 추가 주의사항
- 관리자 읽기용 1분 문안

### 예시 3: TBM 일지
- 작업명 / 일시 / 참석자
- 전달내용
- 확인사항
- 특이사항
- 후속조치

### 발표 멘트 초안
"저희 서비스의 결과물은 검색 결과가 아니라, 현장에서 바로 읽고 쓰고 남길 수 있는 문서입니다."

---

## 7장. 실질적 TBM 유도
### 제목
형식적 TBM이 아니라, 실제 대화를 유도합니다

### 핵심 아이디어
- 단순 일지 자동생성이 아니라
- 질문형 진행으로 팀이 위험을 다시 생각하게 함

### 예시 질문
- 오늘 작업에서 가장 큰 위험은 추락, 끼임, 낙뢰 중 무엇인가?
- 강풍 시 중지 기준을 누가 판단할 것인가?
- 신규 작업자에게 반드시 다시 설명할 포인트는 무엇인가?
- 장비/보호구 점검에서 빠질 수 있는 항목은 무엇인가?

### 발표 멘트 초안
"진짜 목적은 문서를 예쁘게 만드는 것이 아니라, TBM이 실제 위험 대화를 유도하도록 돕는 것입니다."

---

## 8장. 차별점
### 제목
검색형 안전정보 서비스와 무엇이 다른가

### 기존 서비스 한계
- 날씨는 날씨대로, 법령은 법령대로 분리
- 안전관리자가 직접 다시 조합해야 함
- 결과가 산출물로 남지 않음
- 현장 실행과 기록 연결이 약함

### 우리 차별점
- 공공데이터를 한 흐름으로 연결
- 법령 근거 + 실무 조치 동시 제시
- 위험성평가/TBM 산출물 생성
- 실질적 TBM 질문까지 제공

### 발표 멘트 초안
"저희는 정보를 더 많이 보여주는 서비스가 아니라, 현장 판단과 기록까지 이어지는 실행형 서비스입니다."

---

## 9장. 현재 개발 상태와 MVP
### 제목
지금 어디까지 만들었는가

### 현재 상태
- 하네스 v1 구축 완료
- `contest-mvp` 스캐폴드 완료
- `/search`, `/ask`, `/law/[id]`, `/precedent/[id]` 구현
- mock 기반 검색/질문 흐름 구현
- build 통과 확인
- 브리핑/운영 문서화 완료

### MVP 다음 단계
1. 제품 스토리 1개 확정
2. 데모 시나리오 1개 확정
3. 질문형 결과를 `날씨 → 법령 → 현장조치` 구조로 고도화
4. live adapter 연결 우선순위 적용
5. TBM 산출물 출력 포맷 추가

### 발표 멘트 초안
"아이디어 단계가 아니라 이미 MVP 골격은 올라와 있고, 지금은 제출용 데모 시나리오를 다듬는 단계입니다."

---

## 10장. 기대효과와 확장성
### 제목
공공데이터를 현장 안전 운영의 기본 도구로

### 기대효과
- 산업안전 담당자의 판단 속도 향상
- 법령 근거 기반 브리핑 자동화
- TBM의 실질성 강화
- 위험성평가 초안 작성 부담 감소
- 교육/점검/감사 대응 기록 품질 향상

### 확장 가능성
- 업종별 위험성평가 템플릿
- 화학물질/MSDS 교육 지원
- 사고사례 기반 경고 추천
- 협력업체/다국어 현장 브리핑
- 안전점검표 / 교육자료 자동 생성

### 마무리 문장
"저희는 공공데이터를 단순 조회형 서비스가 아니라, 오늘 현장에서 실제로 위험을 줄이는 실무형 안전 코파일럿으로 바꾸고자 합니다."

---

# 부록: 발표 톤 가이드
- `자동 작성`보다 `초안 생성`, `보조`, `유도`라는 표현을 쓴다.
- `법 위반 판단`보다 `관련 법령상 조치 검토 필요`처럼 신중한 문구를 쓴다.
- 심사위원에게는 기술보다 **현장 문제 해결과 산출물 가치**를 먼저 보여준다.
