## 결론

현재 SafeClaw는 핵심 엔진(`/workspace` 문서팩 생성, 근거 결합, 다운로드, 작업자/전파 일부)은 살아 있지만, 디자이너가 정의한 “산업안전 운영 OS” 화면 체계와 아직 완전히 일치하지 않는다.

가장 큰 갭은 세 가지다.

1. `로그인`, `홈 대시보드`, `설정`, `작업자 모바일`, `TBM 풀스크린`은 이름은 있으나 실제 운영 기능으로 닫힌 화면이 아니다.
2. 최종 브랜드 가이드의 로고/워드마크, 다크 HUD 셸, 노랑 사용 규칙, 그리드/타이포 규칙이 전역으로 잠기지 않았다.
3. `/workspace`가 여전히 기존 `SafeGuardCommandCenter` 엔진 중심이라 랜딩·모듈 화면과 시각 언어가 다르다.

## P0로 닫아야 할 것

1. `/login`을 만들거나 “로그인” 버튼명을 “작업공간”으로 바꾼다. 지금처럼 로그인 버튼이 `/workspace`로 가면 명칭과 동작이 다르다.
2. `/settings`를 실제 설정 화면으로 만들거나 “운영 설정 준비”로 명확히 낮춘다.
3. `/archive`에서 저장된 문서팩을 정확히 다시 여는 경로를 만든다. 상용 SaaS의 재방문 가치는 여기서 결정된다.
4. `/workspace`를 SafeClaw 브랜드 셸과 동일한 로고/상단/메뉴/타이포로 감싼다.
5. 로고를 하나로 고정한다. 현재 `SC` placeholder는 디자이너 원본 로고와 맞지 않는다.
6. 운영 화면에서 `mappedTo`, `제품화 중`, `Saved workpacks`, `Last generated`, `Auth UI 후속` 같은 내부 구현 설명을 제거한다.

## P1로 올릴 것

1. `/documents`는 저장된 workpack id 기준으로 문서팩을 다시 열 수 있어야 한다.
2. `/evidence`는 근거 목록보다 “현재 문서 문장에 어떤 근거가 붙었는지”를 우선 보여줘야 한다.
3. Google Sheets CTA는 자동 연동이 아니면 “TSV로 시트에 붙여넣기”로 이름을 바꾼다.
4. HWPX/PDF는 “제출형 초안”과 “원본 셀 단위 복제 아님”을 운영 문구로 분리한다.
5. `/home`, `/worker`, `/tbm`은 예정 화면이라도 실제 대체 경로와 현재 사용 가능한 기능을 더 명확히 보여준다.

## 현재 제출 관점 판정

공모전 제출용으로는 핵심 데모와 공공 API/AI 활용 흐름을 보여줄 수 있다. 다만 “상용 SaaS v1”이라고 주장하려면 로그인/이력/설정/재오픈이 아직 약하다. 디자인도 방향은 SafeClaw로 이동했지만, 디자이너 최종안이 전역 제품 시스템으로 완전히 이식된 상태는 아니다.

추천 순서는 다음과 같다.

1. 명칭과 동작 불일치 제거.
2. 로고/브랜드 셸 통일.
3. `/archive` 재오픈과 `/login` 진입 완성.
4. `/workspace`를 최종 셸로 감싸기.
5. 문서/근거/전파 내부 컴포넌트를 픽셀 패스로 다듬기.
