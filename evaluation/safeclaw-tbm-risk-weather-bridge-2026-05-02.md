# SafeClaw TBM 위험성평가·기상 반영 검증

## 목적
TBM 브리핑과 TBM 기록이 독립 문서처럼 보이지 않도록, 위험성평가표의 주요 유해·위험요인과 기상 API 결과를 TBM 전달 항목에 직접 연결했다.

## 반영 내용
- 위험성평가표에서 추락, 전도, 충돌, 끼임, 화재, 중독, 노출 등 주요 위험 행을 추출한다.
- 기상 API 결과의 조치 문구가 있으면 우선 사용하고, 없으면 기상 요약 또는 현장 브리프의 날씨 정보를 사용한다.
- TBM 브리핑과 TBM 기록에 `위험성평가·기상 API 반영` 표를 추가한다.
- 표 항목은 `주요 유해·위험요인`, `오늘 기상/환경 신호`, `TBM 전달 문구`로 고정했다.

## 출력 경로
- 화면 미리보기: `SafetyDocumentPreview`
- HTML/PDF 인쇄 대체: `buildHtml`
- Excel 제출형: `buildExcelHtml`
- Word 보고서형: `buildWordHtml`
- HWPX 제출형 텍스트: `buildHwpTemplateText`
- JPG 렌더: HTML 렌더 경로를 공유

## 서식 구분
- 위험성평가표: 사전준비, 유해·위험요인 파악, 위험성 결정, 감소대책, 공유·교육, 조치 확인 구조.
- 작업계획서: 작업개요, 작업순서, 장비·인원·첨부서류, 작업중지 기준, 비상대응 구조.
- 안전작업허가 확인서: 허가 기본정보, 허가조건, 첨부서류, 종료 확인 구조.
- TBM 기록: 회의 정보, 공유 내용, 위험성평가·기상 API 반영, 참석자 확인, 미조치·증빙 구조.

## 검증
- `npm.cmd run typecheck`: 통과.
- `npm.cmd run build`: 통과.

## 2026-05-03 추가 보강
- 위험성평가표의 `유해·위험요인 파악 및 위험성 결정` 표에 4M 구분, 재해형태, 가능성, 중대성, 등급, 현재 안전조치를 함께 표시하도록 보강했다.
- 위험성평가표에 `위험성평가 회의록 및 재평가 기준` 섹션을 추가해 공법·장비·인원·기상 조건 변경 시 재평가 흐름을 남기게 했다.
- 작업계획서에 발주자/원청 제출용 첨부서류, MSDS·안전인증·안전검사, 작업계획도, 작업지휘자, 유도자, 운전원, 감시자 지정 항목을 추가했다.
- 안전작업허가 확인서의 첨부 체크리스트에 보험가입증명, 자격/면허, 안전교육 이수증, 작업반경도, 인양능력표, 정비/점검 이력 항목을 추가했다.
- TBM 브리핑은 일반 문서 렌더링 대신 `작업 전 브리핑 개요`, `위험성평가 기반 TBM 전달`, `확인 질문`, `보호구·통제·증빙` 전용 구조로 분리했다.
- TBM 브리핑의 위험 전달 표는 위험성평가표에서 추출한 주요 유해·위험요인과 기상 API 결과를 같은 행에서 보여준다.

## 남은 확인
- 브라우저에서 TBM 브리핑/TBM 기록을 선택한 뒤 `위험성평가·기상 API 반영` 표가 보이는지 수동 확인한다.
- 대표 시나리오 3건에서 기상 API 문구가 각각 다르게 들어오는지 E2E로 재검증한다.
