# SafeClaw TBM 제출 준비도 리포트

생성일: 2026-05-03  
DB 변경: 없음  
담당 축: 고용노동부 정식 TBM 가이드 + TBM 일지 완전 이식

## 1. 공식/사용자 원본에서 확인한 필수 항목

원본 기준 TBM은 단순 브리핑 문장이 아니라 보관 가능한 회의록·일지 구조여야 합니다.

- 기본정보: TBM 일시, 작업날짜 동일 여부, 업체명, 현장명, 작업장소, 작업명, 작업내용, 진행자
- 위험성평가 반영: 위험성평가 실시여부, 주요 유해·위험요인, 중점위험요인, 대책, 조치담당
- 기상 API 반영: 기상 요약, 기상 악화 시 조치, 옥외·고소·장비작업 작업중지 재판단
- 작업중지 기준: 중지 트리거, 판단자, 보고자, 재개 조건
- 근로자 확인: 보호구, 건강상태, 음주·피로, 위험구역 통제 확인
- 참석자 확인: 소속, 직종, 성명, 오전/오후 참석, 구두 복창, 서명
- 신규·외국인·미숙련 안내: 쉬운 한국어, 통역 또는 번역 메시지, 이해 여부 별도 확인, 미이해자 재교육
- 사진·증빙: 작업 전 사진, 보호구 사진, 통제구역 사진, TBM 실시 사진, 사진/영상 파일명, 보관 위치
- 교육시간 인정 검토: 교육 대상, 교육 내용, 교육 시간, 서명, 사진 또는 모바일 기록, 법정교육 대체 단정 금지
- 미조치·후속조치: 미조치 위험요인, 후속조치, 담당자, 확인자

원본 근거는 `evaluation/tbm-reference-ingestion/tbm-reference-ingestion-report.json`에 구조화되어 있습니다. `11 건설현장 TBM 실천 가이드.pdf`는 이미지 기반 PDF라 텍스트 추출은 제한되며, 정확 문구 이식을 위해 OCR 검증이 남아 있습니다.

## 2. 현재 생성물 비교

TBM 브리핑:

- 반영됨: 작업내용, 위험요인, 안전대책, 참석자 확인, 사진·증빙 위치, 기상 API 요약, 신규·외국인·미숙련 안내
- 누락: 정식 참석자 서명표, 작업날짜 동일 여부, 위험성평가 실시여부 체크박스, 미조치 후속조치 표
- 과잉/중복: 교육시간 인정 문구가 길어지면 브리핑 목적이 흐려지므로 짧은 검토 문구만 적합

TBM 기록/TBM 일지:

- 반영됨: 업체명, 작업장소, 작업내용, 참석인원, 위험성평가 실시여부, 근로자 확인, 참석자 확인, 사진·영상 기록, 교육시간 인정 검토
- 누락: 전자서명/첨부파일 실제 저장 연결, 현장소장 결재 이미지, HWPX/Excel/PDF 공통 payload
- 과잉/중복: 긴 법령·KOSHA·사례 appendices는 TBM 본문 품질을 떨어뜨리므로 evidence UI/metadata로 분리해야 함

렌더러:

- 현재 `components/WorkpackEditor.tsx`에는 외부 dirty change로 TBM 일지형 표 구조 개선이 이미 들어와 있습니다.
- 이 패스에서는 해당 dirty 파일을 stage하지 않았습니다. 충돌 없이 확정하려면 별도 canonical TBM schema가 먼저 필요합니다.

## 3. 바로 고친 항목

- `lib/mock-data.ts`: fallback/mock TBM 브리핑과 일지에 위험성평가 연결, 기상, 작업중지 기준, 신규·외국인·미숙련 작업자 이해확인, 사진·증빙 위치, 교육시간 인정 검토 문구를 추가했습니다.
- `lib/search.ts`: TBM 본문에 긴 법령·KOSHA·사례 원문성 블록을 붙이는 대신 `[TBM 필수 반영 체크]`로 주요 위험, 기상 API, 작업중지 기준, 참석자 확인, 신규·외국인·미숙련 안내, 사진·증빙 위치, 짧은 반영 근거만 남기도록 바꿨습니다.

## 4. 제출 가능 수준 판단

현재 TBM은 데모/심사용 제출에는 조건부로 가능합니다. 이유는 핵심 위험성평가, 기상, 작업중지, 참석자 확인, 신규·외국인·미숙련 안내, 사진·증빙 위치가 문서 본문에 들어가기 때문입니다.

다만 유료상품 수준은 아직 아닙니다. 서명·증빙 저장, 원본 HWPX와 같은 일지형 geometry, OCR로 확인한 건설현장 TBM 가이드 문구, HWPX/Excel/PDF 공통 schema가 아직 닫힌 루프로 연결되지 않았습니다.

## 5. 남은 항목

- canonical TBM schema를 `session`, `riskBriefing`, `workerChecks`, `attendance`, `evidence`, `educationRecognition`, `signOff`로 구현
- `components/WorkpackEditor.tsx`의 현재 dirty TBM 일지 렌더러를 충돌 없이 확정하고 HWPX/Excel/PDF export에 동일 payload 적용
- 사진·영상 파일명, 모바일 로그 ID, 참석자 서명, 확인자 결재를 저장 계층과 연결
- `11 건설현장 TBM 실천 가이드.pdf` OCR 후 정식 건설현장 TBM 필드/문구 재검증
- TBM 품질 smoke에 필수 항목 존재뿐 아니라 “위험성평가-기상-작업중지-참석확인-증빙” 연결성 검사 추가
