### 추출 결과

- 총 8페이지이나 텍스트 레이어는 페이지 제목 수준으로만 존재한다.
- 추출된 대표 문구는 `위험성평가 접속 방법`, `위험성평가 신규작성`, `아래 위험성평가 내용 입력 후 클릭` 수준이다.
- 본문 대부분은 시스템 화면 캡처 이미지이므로 OCR 없이는 필드명, 버튼명, 입력값을 안정적으로 파싱하기 어렵다.

### 주요 흐름

- 한국남동발전 KOEN 사이트에서 `E-biz센터`로 진입한다.
- e-Biz center 로그인 후 `위험성평가` 메뉴를 선택한다.
- 수시 위험성평가 지원시스템(KORAS)에서 기존 평가 목록을 확인한다.
- `신규생성하기`로 평가 문서를 생성한다.
- 수시 위험성평가 개요 화면에서 기본정보와 작업정보를 입력한 뒤 저장한다.
- 저장 후 필요 시 수정 버튼으로 재진입한다.

### 화면 기반 입력 필드

- 승인란: 담당, 차장, 부장
- 작업 기본정보: 작업명, 평가 일자, 작업 기간, 소속, 작업 장소, 작업 인원
- 작업 준비정보: 필요한 보호구, 필요한 공기구, 투입 중장비
- 평가 작성정보: 위험성평가자, 작업 절차서
- 작업 분류: 일반작업, 화기작업, 밀폐작업, 고소작업, 정전작업, 굴착작업, 잠수작업, 중장비작업, 방사선작업
- 작업 단계 행: 주요 작업단계, 작업내용, 기인물

### SafeClaw 반영 포인트

- 위험성평가 생성 화면은 `개요 -> 위험성평가표 -> 실시결과/작업자 확인 -> 최종 리포트`처럼 단계형 탭 구조를 갖는다. SafeClaw 문서 생성도 한 번에 긴 문서를 만들기보다 단계별 진행 상태를 보여주는 쪽이 적합하다.
- 기본정보 입력에서 작업명, 기간, 장소, 인원, 보호구, 공기구, 중장비, 작업 분류가 먼저 고정된다. SafeClaw 위험성평가/JSA 생성 전에도 같은 필드를 필수 입력 또는 자동 추론 대상으로 둬야 한다.
- 작업 분류 체크박스는 후속 위험요인 추천의 핵심 조건이다. 예를 들어 화기작업, 밀폐작업, 고소작업 같은 분류가 켜지면 해당 분류별 특화 위험요인과 관리대책을 우선 노출해야 한다.
- `주요 작업단계 / 작업내용 / 기인물` 행은 JSA의 작업 단계 분해와 직접 연결된다. SafeClaw 문서 품질 검증 시 작업 단계가 하나의 문장으로 뭉개지지 않았는지 확인해야 한다.

### 위험성평가 이행점검 항목

교안에는 위험성평가 이행점검을 다음 범주로 정리한다.

- 계획: 위험성평가 실시규정 작성·관리, 사전준비·활용
- 이행: 대상별 유해·위험요인 파악, 위험성 추정·결정, 감소대책 수립 및 개선활동
- 지속관리: 수시·정기평가 실시
- 기록: 위험성평가 기록 및 보존
- 교육: 실시 전 교육, 실시 후 교육
- 수급업체 점검 및 보완조치

SafeClaw 문서 품질 체크도 위 범주를 그대로 검증축으로 삼을 수 있다. 단순히 위험요인을 나열하는 수준이 아니라 실시규정, 사전준비, 기록보존, 교육, 수급업체 보완조치까지 포함해야 완성도가 높다.

### 위험성평가 절차

교안의 전체 흐름은 아래 순서로 정리된다.

- 공정 선정과 사전준비
- 유해·위험요인 파악
- 위험성 추정
- 위험성 결정
- 위험성 감소대책 수립·실행
- 허용 가능 여부 확인
- 위험정보 게시 및 교육

사전준비 단계에는 평가 목적과 방법, 담당자와 책임자의 역할, 평가 시기와 절차, 주지방법, 결과 기록·보존이 포함된다. 방법론 측면에서는 작업표준, 작업절차, 기계·기구·설비 사양서, MSDS(물질안전보건자료), 공정 흐름, 작업 주변 환경, 재해 사례, 안전보건 정보, 건강진단 결과 등을 활용하도록 안내한다.

### 위험성평가표 서식 요소

교안에서 확인되는 위험성평가표의 핵심 필드는 다음과 같다.

- 평가공정
- 평가자
- 평가일시
- 평균 위험도
- 단위작업 내용
- 평가 구분
- 위험요인
- 예상재해
- 현재 안전조치
- 개선 전 위험도: 빈도, 강도, 위험성
- 개선대책
- 코드번호
- 개선 후 위험도: 빈도, 강도, 위험도

다른 예시 서식에서는 공정 대분류, 세부분류, 유해위험요인 분류, 원인, 유해위험요인, 관련 법규/노출기준, 현재 안전조치, 감소대책도 함께 확인된다.

### 위험성 추정/결정 기준

- 위험성 추정은 가능성(빈도)과 중대성(강도)을 조합하는 매트릭스 방식이 중심이다.
- 중대성은 최악의 상황에서 예상되는 부상 또는 질병의 크기를 기준으로 보도록 안내한다.
- 유해성이 완전히 입증되지 않아도 일정한 근거가 있으면 유해성이 존재하는 것으로 보고 추정하도록 한다.
- 위험성 결정은 사업장이 사전에 정한 허용 가능 기준과 비교해 판단한다.
- 허용 가능 기준에 도달하지 못하면 추가 감소대책을 반복 수립·실행해야 한다.

SafeClaw에서는 산출 문서에 수치만 표시하기보다 `판단 근거`, `현재 안전조치`, `개선 전/후 변화`, `추가 조치 필요 여부`를 함께 남겨야 한다. 특히 근거가 불확실한 위험도 무시하지 않고 보수적으로 표시하는 규칙이 필요하다.

### JSA 수행절차

교안의 JSA 설명은 작업을 주요 단계로 나눈 뒤 각 절차별 유해위험요인과 잠재 사고를 파악하고, 이를 제거·최소화·예방하기 위한 대책을 개발하는 방식이다.

핵심 원칙은 다음과 같다.

- 작업자, 작업, 작업도구 또는 장비, 작업환경 사이의 관계에 집중한다.
- 작업이 어떻게 수행될지 관계자 간 의견교환이 가능해야 한다.
- 팀장, 경험 작업자, 직접 수행 작업자 등 실제 작업 관련자가 참여해야 한다.
- 작업자 지식은 위험성을 찾는 데 중요하므로 JSA 작성 과정에 참여시키는 것이 필요하다.
- 작업 순서대로 단계를 구분하되 너무 자세하거나 너무 포괄적으로 나누지 않는다.
- 각 작업별 발생 가능한 위험성을 모두 적는다.
- 각 위험요인에 대한 대책과 안전한 작업방법을 작성한다.
- 작업 수행 전에 대책 실행 여부와 담당자를 확인한다.
- 결과는 문서로 등록하고 관련 작업자에게 교육한다.
- 작업방법 또는 작업내용이 바뀌면 JSA를 다시 실행한다.

### JSA/KRAS 서식 요소

JSA 예시와 KPS/KRAS 서식 이미지에서 확인되는 필드는 다음과 같다.

- 작업명
- 작성자, 검토자, 승인자
- 부서명, 작업지역, 작업공정
- 필요한 개인보호구
- 필요한 장비 및 공구
- 필요한 자료
- 작업단계
- 유해위험요인
- 현재 안전조치 또는 현재 위험도
- 대책 또는 안전한 작업방법
- 개선 후 위험도
- 개선 실행 계획서
- 개선 실행 일자, 내용, 담당자
- 확인 일자
- 비고

JSA 예시는 `작업단계(step) / 유해위험요인(hazards) / 대책(안전한 작업방법)`의 세 열을 중심으로 한다. SafeClaw의 JSA 문서는 이 세 열을 최소 단위로 보장해야 하며, 위험성평가표와 연결할 때는 현재 조치와 개선 후 확인 필드를 추가해야 한다.

## 4. SafeClaw 문서 품질 반영 체크리스트

아래 항목은 SafeClaw 위험성평가/JSA 문서 생성 또는 리뷰 루브릭에 반영할 수 있다.

- 작업 기본정보가 충분한가: 작업명, 기간, 장소, 인원, 소속, 평가일자, 평가자
- 작업 준비정보가 빠지지 않았는가: 보호구, 공기구, 중장비, 작업 절차서
- 작업 분류가 명시되어 있는가: 화기, 밀폐, 고소, 정전, 굴착, 잠수, 중장비, 방사선 등
- 작업 단계가 실제 순서대로 분해되어 있는가
- 각 작업 단계마다 작업내용과 기인물이 구분되어 있는가
- 유해·위험요인이 작업 단계와 직접 연결되어 있는가
- 예상재해가 구체적으로 적혀 있는가
- 현재 안전조치가 적혀 있는가
- 개선 전 위험성과 개선 후 위험성이 구분되어 있는가
- 감소대책이 단순 주의 문구가 아니라 실행 가능한 조치인가
- 개선대책마다 담당자, 실행 시점, 확인 여부가 붙어 있는가
- 허용 가능 여부 판단과 추가 조치 필요 여부가 남아 있는가
- 위험정보 게시와 작업자 교육 계획이 포함되어 있는가
- 수급업체가 관련된 작업이면 점검과 보완조치 항목이 포함되어 있는가
- 작업방법 또는 작업내용 변경 시 재평가 조건이 명시되어 있는가
- 기록 보존 항목이 포함되어 있는가

## 5. 파싱 전략 제안

- `위험성평가 접근방법 메뉴얼.pdf`는 OCR 대상이다. 현재 텍스트 레이어만으로는 메뉴얼 필드 자동 수집에 부적합하다.
- `위험성평가 JSA 교육 교안.pdf`는 텍스트 추출과 이미지 OCR을 혼합해야 한다. 절차 설명은 텍스트 추출로, 서식 표는 렌더링 후 OCR/수동 템플릿화로 처리하는 것이 안전하다.
- 슬라이드 레이아웃 표와 실제 업무 서식 표가 섞이므로 pdfplumber 표 결과를 그대로 데이터로 쓰면 안 된다.
- 우선 수동으로 표준 스키마를 만든 뒤, PDF에서 확인된 필드를 스키마 검증 근거로 사용하는 방식이 적합하다.
- SafeClaw의 표준 스키마 후보는 `work_info`, `preparation`, `task_classification`, `job_steps`, `hazards`, `current_controls`, `risk_before`, `controls_to_improve`, `risk_after`, `implementation_check`, `worker_training`, `recordkeeping`으로 나눌 수 있다.

## 6. 결론

두 PDF는 원문 전체를 안정적으로 기계 파싱하기보다는 SafeClaw 위험성평가/JSA 산출물의 구조와 품질 검증 기준을 만드는 근거 자료로 쓰는 편이 적합하다. 특히 KOEN/KORAS 메뉴얼은 입력 플로우와 필드명 참고용, JSA 교육 교안은 위험성평가 절차·JSA 단계·평가표/KRAS 서식 품질 기준 참고용으로 가치가 높다.

SafeClaw에 즉시 반영할 핵심은 다음 세 가지다.

- 위험성평가 문서를 `기본정보 -> 작업단계 -> 위험요인 -> 현재조치 -> 개선대책 -> 개선 후 확인 -> 교육/기록` 흐름으로 강제한다.
- JSA는 `작업단계 / 유해위험요인 / 안전한 작업방법`을 최소 필수 열로 두고, 실제 작업자 참여와 작업 변경 시 재평가 문구를 포함한다.
- 산출물 리뷰는 단순 내용량보다 현재 조치, 개선 전후 비교, 담당자/확인일, 교육/게시/기록 보존 여부를 기준으로 판정한다.
