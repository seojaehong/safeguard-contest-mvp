## Data Quality Notes

- `기계설비명` 기준 중복은 4개 명칭에서 확인됐다. 중복 명칭은 업종소분류가 다르므로 단순 dedupe하면 업종 맥락이 손실된다.
- 중복 설비명: `루터기`, `유압프레스`, `전동 드릴`, `휴대용 연삭기`.
- `기계설비설명`은 문장 중 쉼표 대신 `/`를 나열 구분자로 쓰는 경우가 많다. 화면 표시나 검색 색인 단계에서는 `/`를 쉼표 또는 bullet로 정규화하면 가독성이 좋아진다.
- `업종대분류`, `업종중분류`, `업종소분류`, `기계설비명`의 조합을 SafeClaw 내부 식별자 생성 기준으로 삼는 것이 안전하다.
- CSV 파서는 UTF-8 자동 가정이 아니라 `cp949` 우선, 실패 시 `euc-kr` fallback을 둬야 한다.

| `기계설비설명` | 작업 범위와 설비 특성 | 자동공구교환장치 포함, 토치 이동 절단, 내연기관 동력 등 |

### 작업계획서 Template Hooks

- `equipment.identity`: 업종분류 + 설비명 + 영문명 + 원천 순번
- `equipment.operation_summary`: `기계설비설명`을 짧게 정규화한 작업 개요
- `equipment.hazard_groups`: 룰 기반 위험요인 그룹 배열
- `work_boundary`: 운전, 정비, 청소, 공구 교체, 반입·반출 등 작업 단계
- `required_controls`: 위험요인 그룹별 필수 통제조치
- `pre_start_checklist`: 방호장치, 비상정지, 누유·누압, 전원차단, 보호구, 작업반경 통제
- `permit_flags`: 화기, 고소, 양중, 밀폐공간, 정비잠금 등 허가/승인 필요 여부

## Suggested Ingestion Shape

```ts
type MachinerySourceRecord = {
  sourceId: string;
  sourceFile: string;
  rowNumber: number;
  industryMajor: string;
  industryMiddle: string;
  industryMinor: string;
  equipmentNameKo: string;
  equipmentNameEn: string;
  equipmentDescription: string;
  normalizedDescription: string;
  hazardGroups: string[];
};
```

### Normalization Rules

- `sourceId`: `kosha-machinery-20210909-${번호}` 형태로 생성한다.
- `normalizedDescription`: `/`를 문장 구분 또는 쉼표로 치환하고 중복 공백을 제거한다.
- `hazardGroups`: 장비명, 영문명, 설명 전체를 대상으로 키워드 매칭한다.
- `industryPath`: `업종대분류 > 업종중분류 > 업종소분류` 형태로 별도 display field를 만든다.
- `equipmentNameKo` 단독으로 dedupe하지 말고 `industryPath + equipmentNameKo` 기준으로 유지한다.

## SafeClaw Product Recommendations

- 검색/추천: 사용자가 업종만 선택해도 해당 업종의 대표 설비와 위험요인을 추천할 수 있다.

- 현장 점검 UI: 같은 장비명이라도 업종소분류가 다르면 표시 설명과 점검 항목을 분리한다.
