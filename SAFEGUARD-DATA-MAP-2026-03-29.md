# SafeGuard 공공데이터 활용 매핑표

## 목적
심사위원이 한눈에 "어떤 데이터를, 왜, 어떻게 쓰는지" 이해할 수 있도록 정리한 제출 보조 문서.

| 분류 | 데이터/API | 역할 | 산출물 연결 |
|---|---|---|---|
| 기상 신호 | 기상청 단기예보 | 온도/기본 기상 상태 파악 | 오늘 작업 환경 요약 |
| 기상 경보 | 기상청 기상특보 | 강풍/호우/건조/폭염 특보 감지 | 위험도 상승, 작업중지 검토 |
| 건강영향 | 기상청 영향예보 | 폭염/한파 건강영향 보강 | 폭염/한파 조치 강화 |
| 법령 근거 | Law.go / 국가법령정보 | 법령/조문 인용 | 관련 법령 블록 |
| 안전 법령 검색 | KOSHA 안전보건법령 스마트검색 | 안전보건 관련 법령 탐색 | 근거/설명 보강 |
| 사고 사례 | KOSHA 국내재해사례 | 유사 사고 연결 | TBM 경고 포인트 |
| 사고사망 정보 | KOSHA 사고사망 사례 | 경각심/실제 사례 요약 | 교육/브리핑 포인트 |
| 화학물질 | KOSHA MSDS | 규제·유해성 정보 파악 | 물질 취급 주의사항 |
| 자료 링크 | KOSHA 안전보건자료 링크 | 관련 가이드/자료 제시 | 후속 학습/증빙 보강 |

## 핵심 설명
- 기상청은 **위험 신호**를 준다.
- KOSHA는 **실무 안전 데이터와 가이드**를 준다.
- 법령 API는 **판단의 근거**를 준다.
- 세이프가드는 이 3축을 묶어 **위험성평가·TBM 산출물**로 바꾼다.
