# SafeClaw 공공기관 안전서류 루브릭 적용 기준

## 목적

SafeClaw의 문서팩을 단순 생성물이 아니라 `제출 전 점검`이 가능한 산출물로 만들기 위한 기준입니다. 사용자가 제공한 공공기관형 안전서류와 일반 안전점검표 구조에서 공통 요소만 추출해, 특정 발주처 전용 문구가 아닌 공통 안전서류 품질 기준으로 재구성했습니다.

## 2트랙 운영

| 트랙 | 용도 | 제품 표현 |
|---|---|---|
| 내부 품질검수 | 생성된 문서팩이 필수 항목을 담았는지 자동 점검 | 내부 루브릭 |
| 사용자 제출 전 점검 | 사용자가 보완할 항목을 쉽게 확인 | 제출 전 점검 |

## 기준 분류

| 분류 | 의미 | 예시 |
|---|---|---|
| 필수 확인 | 법령, 고시, 공식자료와 직접 연결되는 확인 항목 | 유해·위험요인, 감소대책, 작업중지 기준, 근로자 공유 |
| 제출 품질 보강 | 공공기관·발주처형 안전서류에서 자주 확인하는 품질 항목 | 확인자, 서명란, 개선요구, 조치결과, 증빙 위치 |
| 현장 운영 추천 | 현장 실행력을 높이는 보조 항목 | 외국인 전송본, 쉬운 한국어, 작업자 회신, 사진 증빙 |
| ISO 운영체계 | ISO 45001식 안전보건경영시스템으로 확장하기 위한 운영 항목 | 법규 요구사항 등록부, 문서관리, PDCA, 감사 추적성, 개선조치 |

## ISO 45001 관점의 해석

SafeClaw의 목표 수준은 단순 문서 생성이 아니라 `Plan-Do-Check-Act`가 현장 작업 단위로 닫히는 안전보건 운영 보조입니다.

| ISO 45001 관점 | SafeClaw 반영 | 대표 문서 |
|---|---|---|
| Plan | 작업 전 위험성평가, 작업계획, 법규 요구사항 확인 | 위험성평가표, 작업계획서, 근거 목록 |
| Do | TBM, 안전보건교육, 작업중지 기준, 외국인 전달 | TBM 브리핑, TBM 기록, 안전보건교육 기록, 외국인 전송본 |
| Check | 조치 이행 확인, 사진/증빙, 교육 확인, 전파 결과 | 점검결과 요약, 사진/증빙, 전파 로그 |
| Act | 미조치 항목, 개선조치, 재발방지, 다음 작업 반영 | 비상대응 절차, 후속조치, 개선조치 관리 |

제품 표현에서는 `ISO 45001 인증 대체`나 `법적 효력 보장`으로 말하지 않습니다. 대신 `ISO 45001식 문서체계에 맞춘 운영 보조`, `공식자료 기반 초안`, `현장 검토 후 제출 가능한 문서팩`으로 표현합니다.

## 사용자 경험 원칙

- 수치 평가 대신 `충족`, `보완 필요`, `사용자 확인 필요`로 표시합니다.
- `필수 확인`과 `제출 품질 보강`을 분리해 법적 의무처럼 과장하지 않습니다.
- 미충족 항목은 `보완 생성`과 `리서치` 액션으로 이어집니다.
- 전체 Excel export에는 `7. 제출 전 점검` 섹션을 포함합니다.

## 다음 보강 대상

- 보완 생성 버튼을 문서별로 연결합니다.
- 지식 DB에서 항목별 법령·KOSHA 근거를 더 직접 연결합니다.
- 사진/증빙 업로드가 붙으면 `사진·증빙 보관 위치` 항목을 실제 파일 로그와 연결합니다.
- ISO 운영체계 항목은 `문서관리`, `감사 추적성`, `개선조치`를 실제 DB 이력과 연결합니다.
