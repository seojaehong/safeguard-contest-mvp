# SafeClaw 고객사 서식 렌더링 계획

## 목적

SafeClaw의 기본 문서팩은 표준 안전서식 흐름을 빠르게 생성하는 데 맞춰져 있다. 유료 파일럿에서는 여기서 한 단계 더 나아가, 사업장이 이미 쓰는 위험성평가표, 작업계획서, TBM 일지, 안전보건교육 기록 양식을 반복 렌더링할 수 있어야 한다.

이 문서는 현재 구현과 유료 제품화를 분리해 설명한다. 현재 화면은 표준 SafeClaw 서식을 제공하고, 고객사 원본 서식은 온보딩 단계에서 업로드·매핑·검수한 뒤 사용한다.

## 현재 지원 범위

- 표준 Excel 시트형: 표지, 섹션 요약, 확인 칸, 인쇄 폭 기준.
- 표준 Word 보고서형: 제목, 메타, 본문형 설명, 표 혼합.
- 표준 HWPX 제출형: 한글에서 열기 쉬운 제목, 섹션, 확인·서명란.
- 문서별 편집: 위험성평가표, 작업계획서, 작업허가서, TBM, 교육기록 등.
- 보완 문구: 법령, KOSHA, 내부 지식 DB 근거를 보고 제안 후 사용자가 편집·삽입.

## 고객사 서식 렌더링 방향

1. 원본 서식 수집
   - 고객사 XLSX/HWPX 원본을 온보딩 때 받는다.
   - 직인, 결재선, 회사명, 현장명, 표 머리글, 안전관리 항목을 분리해 확인한다.

2. 필드 매핑
   - 현장명, 작업명, 작업자 수, 주요 유해·위험요인, 감소대책, 교육내용, 확인자·승인자 필드를 SafeClaw 문서팩과 연결한다.
   - 매핑된 필드는 한 번 검수한 뒤 같은 사업장·현장에 반복 적용한다.

3. 렌더링 검수
   - 원본 서식의 모든 셀을 자동 복제한다고 표현하지 않는다.
   - 제출 전 사람이 열어 표 깨짐, 페이지 넘김, 서명란, 항목 누락을 확인한다.
   - 검수된 템플릿만 고객사 제출본으로 표시한다.

4. 반복 사용
   - 같은 현장은 전날 문서팩, 작업자 명단, 교육 이력, 전파 로그를 재사용한다.
   - 오늘 바뀐 작업 조건만 다시 반영해 문서팩을 재생성한다.

## 공통 매핑 필드

| 필드 | SafeClaw 원천 | 반영 위치 |
| --- | --- | --- |
| 현장명 | `scenario.siteName` | 표지, 작업개요, 결재란 |
| 작업명 | 정제된 작업내용 | 작업계획서, TBM, 교육기록 |
| 주요 유해·위험요인 | 위험성평가표 행 | 위험성평가표, TBM |
| 감소대책 | 즉시조치, 통제대책 | 위험성평가표, 작업허가, TBM |
| 작업자·교육대상 | 작업자 명단 | TBM 기록, 안전보건교육 기록 |
| 교육내용 | 안전보건교육 기록 | 교육일지, 외국인 안내문 |
| 확인자·승인자 | 사용자 입력 | 결재란, 서명란 |

## 제출 기준 표현

- 사용 가능: “표준 SafeClaw 제출형으로 출력합니다.”
- 사용 가능: “고객사 원본 서식은 온보딩 후 매핑해 반복 출력할 수 있습니다.”
- 금지: “모든 공공기관 원본 서식을 완벽히 복제합니다.”
- 금지: “법적 효력을 보장합니다.”

## 후속 구현 게이트

- XLSX 원본 파서와 셀 주소 매핑 저장소.
- HWPX 원본 템플릿 필드 치환기.
- 템플릿별 QA 리포트: 파일 크기, 한글 깨짐, 서명란, 주요 섹션 포함 여부.
- 관리자 UI: 템플릿 업로드, 필드 매핑, 검수 완료, 사용 중지.
