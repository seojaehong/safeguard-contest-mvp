# SafeGuard Launch Readiness Report

## 기준

1차 런칭 기준은 `오늘 작업 -> 문서팩 생성 -> 근거 확인 -> 수정/다운로드 -> 현장 전파`가 끊기지 않는 것입니다. 사용자 화면에서는 내부 구현 상태어 대신 `연결됨`, `일부 근거 보류`, `연결 점검 필요`로만 표시합니다.

## 이번 마감 반영

- 첫 화면과 보조 화면의 공모전·데모성 문구를 제품형 문구로 정리했습니다.
- 문서팩 산출물에 `점검결과 요약`, `작업계획서`, `비상대응 절차`, `사진/증빙`을 추가했습니다.
- XLS/CSV/TSV 출력 구조를 현장 서식형 시트 구성으로 재정렬했습니다.
- 외국인 근로자 전송본은 언어 선택 후 같은 메시지가 미리보기와 전파 payload에 들어가도록 유지했습니다.
- n8n 전파 payload에 신규 문서팩과 외국인 메시지, 근거, 재해사례가 함께 포함됩니다.

## 서식 구조

다운로드용 시트 구조는 다음 순서로 고정합니다.

1. `0. 문서팩 요약`
2. `1. 위험성평가`
3. `2. 작업계획서`
4. `3. TBM 및 일일점검`
5. `4. 안전보건교육`
6. `5. 비상대응`
7. `6. 사진/증빙`

## 남은 리스크

- Vercel에서 n8n 전파를 직접 사용하려면 `N8N_PUBLIC_BASE`가 공개 HTTPS relay로 설정되어야 합니다.
- 문자·카카오·밴드 발송은 n8n 워크플로우 쪽 채널 credential과 템플릿 상태에 영향을 받습니다.
- 법령·판례 상세 원문 품질은 Law.go 응답 구조와 검색어 적합도에 따라 달라지므로, 발표 전 대표 입력으로 한 번 더 고정 검증해야 합니다.
