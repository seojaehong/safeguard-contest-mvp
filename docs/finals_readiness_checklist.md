# SafeGuard 발표 전 최종 점검표

## 한 줄 소개
오늘 작업 설명 한 줄을 위험성평가·TBM·안전교육 기록으로 바꾸는 산업안전 코파일럿

## 시연 점검
- 홈 첫 화면에서 문제와 해결이 한 문장으로 보인다.
- 대표 시나리오 입력 1회로 위험요약, 위험성평가, TBM, 안전교육 기록이 모두 생성된다.
- 근거 카드와 상태 배지가 동시에 보인다.
- 카카오 메시지까지 이어져 현장 전파 흐름이 끊기지 않는다.
- 데모 중 실패 시 fallback 문구가 어색하지 않다.

## 데이터 점검
- Law.go live 동작을 최근 산출물로 증명할 수 있다.
- Gemini live 동작을 최근 산출물로 증명할 수 있다.
- 기상청 live 동작을 최근 산출물로 증명할 수 있다.
- 고용24 교육 연계가 안전교육 기록 초안에 실제 반영돼 있다.
- KOSHA는 현재 fallback 보강 축이라는 설명이 문서와 발표에서 일치한다.

## 발표 점검
- 첫 30초 안에 문제와 가치가 끝난다.
- `문제 -> 제품 -> 데이터 -> 시연 -> 효과` 순서가 유지된다.
- `검색`보다 `실행 문서 생성` 메시지가 더 많이 반복된다.
- 외국인 근로자/교육 연계는 확장성이 아니라 현재 흐름 일부로 설명된다.
- 데이터 상태를 과장하지 않는다.

## 문서 점검
- 사업계획서의 데이터 명세와 실제 데모가 충돌하지 않는다.
- 가점 후보 데이터 문구가 모든 문서에서 같다.
- 제품 한 줄 소개가 모든 문서에서 같다.
- 발표자료, 스크립트, 체크리스트, 상태표가 같은 용어를 쓴다.

## 최우수 도전 판정
아래 다섯 조건이 모두 충족되면 `최우수 도전 가능 상태`로 본다.
- 대표 시나리오 데모가 끊기지 않는다.
- 공식 평가항목 다섯 개 중 취약 항목이 없다.
- 데이터 사용 증빙과 발표 멘트가 충돌하지 않는다.
- 가점 대상 데이터 여부가 문서로 정리돼 있다.
- 심사위원용 자료에서 실행 문서 생성형 서비스로 읽힌다.
