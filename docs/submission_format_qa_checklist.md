# SafeGuard 제출 서식 QA 체크리스트

제출용 정식 형식은 `PDF`, `HWPX`, `XLS`로 고정한다. `DOC`, `TXT`, `JSON`, `CSV`, `HTML`, `JPG`는 고급 형식이며 제출 차단 기준에는 포함하지 않는다.

## 공통 원칙

- 모든 출력물은 `공식자료 기반 초안`이며, 현장 검토 후 사용한다.
- 위험성평가, TBM, 안전보건교육, 외국인 전송본에는 현장명, 작업명, 핵심 위험, 즉시 조치가 빠지면 안 된다.
- 법령, KOSHA 자료, 기상, 교육 추천, 재해사례는 문서 본문 또는 확인 근거에 연결되어야 한다.
- 한글, 숫자, 서명란, 확인란이 깨지거나 잘리면 제출 전 수정 대상으로 둔다.

## 대표 시나리오

| ID | 현장 | 검수 포인트 |
|---|---|---|
| `seoul-construction-windy` | 서울 건설 강풍 | 이동식 비계, 추락, 강풍 작업중지, 신규 투입자 교육 |
| `incheon-logistics-rain` | 인천 물류 우천 | 젖은 바닥, 지게차 동선, 보행자 분리, 숙련자 중심 운영 |
| `ansan-manufacturing-foreign-hotwork` | 안산 제조 화기 외국인 포함 | 용접·절단, 화재감시, 환기, 다국어 전송본 |

## PDF 확인

- 브라우저 인쇄 또는 저장 흐름에서 제출본 안내가 보인다.
- 대표 문서 3종에 `위험성평가`, `TBM`, `안전보건교육` 제목이 보인다.
- 확인자, 관리감독자, 서명 또는 확인 칸이 보인다.
- 페이지가 과도하게 잘리거나 카드 배경만 출력되지 않는다.
- 면책 문구가 작게 숨지 않고 확인 가능한 위치에 있다.

## HWPX 확인

- 한컴오피스에서 파일이 열린다.
- 한글이 깨지지 않는다.
- 위험성평가표, TBM 기록, 안전보건교육 기록의 항목명이 유지된다.
- 확인자, 관리감독자, 서명란 또는 확인란이 유지된다.
- 외국인 전송본의 주요 언어가 네이티브 스크립트로 보인다.

## XLS 확인

- 표지 또는 요약 시트가 가장 먼저 보인다.
- 위험성평가, 작업계획서, TBM 및 일일점검, 안전보건교육, 비상대응, 사진/증빙 구조가 보인다.
- 섹션 헤더가 굵게 구분되고, 확인 칸 또는 서명 칸이 있다.
- 인쇄 폭이 한 페이지 기준으로 지나치게 넘치지 않는다.
- Google Sheets에 업로드해도 주요 한글 문구가 깨지지 않는다.

## 현장 검수 질문

- 이 문서를 작업반장이 아침 TBM에서 그대로 읽을 수 있는가?
- 신규 투입자 또는 외국인 근로자에게 별도 설명 없이 전달해도 핵심 위험이 보이는가?
- 원청 또는 내부 관리자에게 제출할 때 “초안”과 “현장 확인 필요”가 명확한가?
- API 근거가 단순 목록이 아니라 문서 문장과 연결되어 보이는가?

## 자동 검증 산출물

제출 전 아래 명령으로 자동 검증 파일을 생성한다.

```powershell
npm.cmd run smoke:submission
```

생성 파일:

- `evaluation/submission-readiness/prod-storage-smoke.json`
- `evaluation/submission-readiness/prod-dispatch-smoke.json`
- `evaluation/submission-readiness/document-format-verification.json`
- `evaluation/submission-readiness/submission-readiness-summary.json`
