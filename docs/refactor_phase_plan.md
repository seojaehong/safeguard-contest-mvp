# SafeGuard Refactor Phase Plan

## 기준

전체 로드맵은 5개 Phase로 나눕니다. 각 Phase는 이전 Phase의 데이터 구조와 사용자 흐름을 깨지 않는 방식으로 누적합니다. 외부 API 호출은 그대로 두고, 수신 이후 레이어만 확장합니다.

## Phase 1: 재방문 동기

목표는 사용자가 같은 현장을 다음 날 다시 여는 이유를 만드는 것입니다.

구현 항목:

- Site를 사용자의 기본 작업 단위로 표시합니다.
- DailyEntry를 도입해 일자별 작업 입력, API 스냅샷, 문서팩 결과를 묶습니다.
- 워크스페이스 상단에 `마지막 작성`과 `어제 대비 변경`을 표시합니다.
- 자유 입력은 유지하되, 기본 모드는 “오늘 바뀐 점 한 줄”로 바꿉니다.
- 모든 문서 푸터에 굵은 면책 고지를 추가합니다.

재사용:

- 기존 `sites`, `workpacks`, `workers`, `education_records`, `dispatch_logs`.

신규 필요:

- `daily_entries`
- `shared_context` JSON 필드 또는 별도 테이블

## Phase 2: 품질과 SSOT

목표는 11종 문서를 각각 고치는 불편을 제거하는 것입니다.

구현 항목:

- SharedContext 편집 패널을 도입합니다.
- 현장명, 작업명, 작업자 수, 신규/외국인 수, 날씨, 위험요인, 통제대책을 한 곳에서 수정합니다.
- SharedContext 수정 시 영향받는 문서 카드에 변경 뱃지를 표시합니다.
- 위험성평가표, TBM, 외국인 전송본을 1급 카드로 확대합니다.
- 나머지 8종 산출물과 보조 내보내기 형식은 접힘 상태로 둡니다.
- Excel과 HWPX는 기본 출력으로 유지하고, 보조 형식은 고급 내보내기로 이동합니다.

신규 필요:

- `shared_context`
- 문서별 derived preview 상태
- 변경 영향 추적 로직

## Phase 3: 증빙과 감사 대응 기록

목표는 “작성했다”가 아니라 “확인하고 남겼다”를 증명하는 것입니다.

구현 항목:

- EvidenceLibrary를 도입해 사진, KOSHA 자료, 재해사례, 교육 확인, 전파 결과를 관리합니다.
- 각 증빙은 위험성평가표, TBM, 안전보건교육 기록, 사진/증빙 중 최소 하나의 반영 위치를 가져야 합니다.
- AuditTrail을 append-only로 남깁니다.
- 봉인 PDF 초안은 해시와 생성 시각을 포함하되, 공식 효력을 확정하는 듯한 문구는 쓰지 않습니다.
- 외국인 근로자 회신 수집은 문자/카카오 승인 상태에 맞춰 단계적으로 붙입니다.

신규 필요:

- `evidence_items`
- `audit_trails`
- `sealed_documents`

## Phase 4: 팀과 결재

목표는 개인 파일럿에서 팀 운영으로 확장하는 것입니다.

구현 항목:

- 작성자, 검토자, 승인자 역할을 분리합니다.
- 승인된 문서만 봉인 PDF 초안 발행 버튼을 활성화합니다.
- 결재선 우회 시도는 AuditTrail에 남깁니다.
- 작업자·교육·전파 로그는 Site와 DailyEntry 단위로 묶어 봅니다.

신규 필요:

- `approval_chains`
- `approval_events`
- 역할 기반 UI 상태

## Phase 5: 본사와 리포트

목표는 다현장 운영과 월간 보고까지 확장하는 것입니다.

구현 항목:

- MultiSiteDashboard를 도입합니다.
- 현장별 상태를 정상, 확인 필요, 즉시 점검으로 표시합니다.
- 월간 리포트는 TBM 실시, 위험성평가 갱신, 외국인 교육 확인, 전파 결과를 한 장으로 요약합니다.
- Webhook은 Slack, Teams, 그룹웨어, n8n relay 순서로 확장합니다.

신규 필요:

- `monthly_reports`
- `site_health_snapshots`
- `webhook_subscriptions`

## Phase 종료 조건

각 Phase는 다음 조건을 통과해야 종료합니다.

- 기존 11종 문서 생성이 깨지지 않습니다.
- 기상청, Law.go, Work24, KOSHA 호출부 변경이 없습니다.
- 카드에서 편집 탭으로 이동하는 흐름이 유지됩니다.
- 작업내용 정제와 4M 체크 차별화가 유지됩니다.
- 다국어 출력본의 원어 스크립트가 유지됩니다.
