# SafeGuard 발표 슬라이드 10장 구성안

## 1. 문제 정의
- 작업 전 위험은 알지만 문서화와 공유가 늦어지는 현장
- 추락, 강풍, 지게차 동선 같은 고위험 작업의 즉시 판단 필요

## 2. 왜 지금 필요한가
- 중소 현장에서는 안전담당 전담 인력과 문서 작성 시간이 부족
- 신규 투입자와 외국인 근로자 대응까지 한 번에 요구됨

## 3. 해결책 한 줄
- 오늘 작업 설명 한 줄을 위험성평가·TBM·안전교육 기록으로 바꾸는 안전 코파일럿

## 4. 핵심 사용자 흐름
- 작업 설명 입력
- 위험 요약과 즉시 조치 확인
- 위험성평가 초안 생성
- TBM 브리핑/일지 생성
- 안전교육 기록 초안 생성

## 5. 데모 화면 포인트
- 오늘 위험 3개
- 즉시 조치
- 법령 근거/출처
- 상태 표시 live / partial / mock / fallback

## 6. 실무 산출물 체인
- 위험성평가표(초안)
- 작업 전 안전점검회의(TBM) 브리핑
- TBM 일지
- 당일 안전교육 기록

## 7. 차별성
- 단순 검색형이 아니라 문서 생성형 안전 코파일럿
- 작업 전 대화를 유도하는 질문 3개 제공
- 근거와 상태를 숨기지 않는 설계

## 8. 공공데이터와 확장성
- 법령/판례/공공데이터 근거 활용
- 외국인 근로자용 쉬운 문장 브리핑, 다국어/TTS, 교육기관 정보 연계 확장

## 9. 현재 검증 상태
- Gemini live 응답 확인
- timeout, retry, graceful fallback 적용
- 일부 공공데이터 경로는 partial 또는 mock으로 투명하게 안내

## 10. 기대 효과
- 작업 전 판단 시간 단축
- 문서 누락 감소
- 교육과 브리핑을 연결한 예방 중심 안전관리
