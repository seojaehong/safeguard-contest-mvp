# SafeGuard 최우수 도전용 심사 루브릭

## 한 줄 소개
오늘 작업 설명 한 줄을 위험성평가·TBM·안전교육 기록으로 바꾸는 산업안전 코파일럿

## 목적
이 문서는 SafeGuard의 현재 상태를 공모전 공식 평가항목에 맞춰 점검하고, 최우수 도전 기준으로 무엇을 더 보강해야 하는지 판단하기 위한 내부 기준서다.

## 공식 평가항목 기준
공식 평가항목은 `완성도`, `데이터·AI 활용성`, `실용성`, `차별성`, `확산가능성`으로 본다.

판정 단계는 아래 다섯 단계로 통일한다.
- 결격
- 취약
- 보통
- 강함
- 압도적

최우수 도전 기준은 다음과 같이 둔다.
- 다섯 항목 중 최소 셋이 `압도적`
- 나머지 항목은 최소 `강함`
- `결격` 항목 없음

## 사전 통과 게이트
아래 다섯 조건은 루브릭 채점 전에 반드시 충족해야 한다.
- 고용노동부 또는 주관·후원기관 공공데이터를 실제 사용하고 문서에 명시한다.
- AI 사용이 제품 핵심 흐름에 실제로 연결돼 있다.
- 웹 URL, 화면 캡처, 발표 흐름이 준비돼 있다.
- 데이터별 `live / fallback / planned` 상태를 숨기지 않는다.
- 가점 데이터 사용 문구를 데이터 명세서, 사업계획서, 발표자료에서 동일하게 유지한다.

## 현재 SafeGuard 잠정 평가
| 항목 | 현재 수준 | 판단 근거 | 최우수 도전 보강 포인트 |
| --- | --- | --- | --- |
| 완성도 | 강함 | 입력 1회로 위험요약, 위험성평가, TBM, 안전교육 기록, 카카오 메시지까지 이어짐 | KOSHA 축을 더 강한 데이터 근거 축으로 정리 |
| 데이터·AI 활용성 | 강함 | Law.go, Gemini, 기상청, 고용24 live 사용과 상태 노출이 가능함 | 데이터-기능 매핑표를 발표자료에 명시 |
| 실용성 | 강함 | 현장 문서 초안과 질문형 가이드가 실제 업무 흐름과 맞음 | 작업 전 준비부터 메시지 전파까지 1일 흐름으로 압축 제시 |
| 차별성 | 강함 | 검색형이 아니라 실행 문서 생성형 구조 | 모든 문서와 발표에서 `실행 문서 생성` 메시지 반복 |
| 확산가능성 | 보통~강함 | 건설 외 4개 업종 시나리오와 교육 연계 가능성 보유 | 외국인 근로자/교육 연계를 사회적 약자 배려와 확장성 축으로 명확화 |

## 심사위원 10초 이해 루브릭
아래 네 질문에 모두 `예`여야 한다.
- 첫 화면에서 무엇을 해결하는지가 한 문장으로 보이는가
- 입력 1회로 결과 4종 이상이 동시에 생성되는가
- 실데이터 사용 여부가 신뢰를 깎지 않게 설명되는가
- 왜 지금 필요한지가 사고예방 관점에서 바로 보이는가

## 증빙 루브릭
아래 항목이 모두 준비돼 있어야 한다.
- API별 상태표
- 가점 대상 데이터 출처와 사용 위치 문서화
- `ask-live.json`, `scenario-e2e`, live integration report 보유
- 사업계획서의 데이터 명세와 실제 데모 일치

## 최우수권 분기 루브릭
아래 네 항목 중 셋 이상이 강하게 충족돼야 한다.
- KOSHA가 단순 링크 보강이 아니라 데이터 근거 축으로 설명된다.
- 외국인 근로자/교육 연계가 현재 흐름 안에 붙어 있다.
- 발표 3분 안에 `문제 -> 제품 -> 데이터 -> 시연 -> 효과` 흐름이 끊기지 않는다.
- 검색 서비스가 아니라 실행 문서형 코파일럿이라는 인상이 남는다.

## 발표에서 그대로 쓸 문장
- SafeGuard는 검색 결과를 보여주는 서비스가 아니라, 오늘 작업을 바로 실행 가능한 안전 산출물로 바꾸는 도구입니다.
- 공공데이터와 AI를 결합해 위험요약, 위험성평가, TBM, 안전교육 기록까지 한 번에 연결합니다.
- 데이터 상태는 live, fallback, planned로 구분해 과장하지 않고 설명합니다.
