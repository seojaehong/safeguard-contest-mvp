# SafeGuard 공모전 접수 입력값 정리

## AI 활용 여부

- AI 활용 여부: `사용`
- 활용 AI명: `제미나이`
- 보조 표기: OpenAI/ChatGPT는 장애 시 fallback 또는 검증 보조로만 사용한다. 접수 화면에서 복수 선택이 가능하면 `제미나이`, `챗GPT`를 함께 선택할 수 있지만, 핵심 AI는 Gemini로 설명한다.

## AI 활용 설명

SafeGuard는 현장 작업 설명, 기상·법령·교육·재해사례 데이터를 조합해 위험성평가, TBM, 안전보건교육 기록, 외국인 근로자 안내문을 자동 생성한다. AI는 공공데이터 원문을 대체하지 않고, 법령·기상·KOSHA·고용24 근거를 문서별 조치 문장과 다국어 안전 공지로 변환하는 역할을 한다.

## 서비스 설명 1000 Byte 이내

SafeGuard는 현장관리자가 오늘 작업 내용을 한 줄로 입력하면 기상청, 법제처, KOSHA, 고용24 등 공공데이터와 AI를 조합해 위험성평가표, TBM, 안전보건교육 기록, 외국인 근로자 안내문, 메일·문자 전파 메시지를 생성하는 산업안전 문서팩 서비스입니다. 실시간 기상 위험, 법령 근거, 유사 재해사례, 후속 교육 정보를 문서별 근거로 연결해 작업 전 판단과 교육·전파를 한 흐름으로 지원합니다.

## 구현 방식 설명에 넣을 문장

SafeGuard는 웹앱에서 문서팩을 생성하고, Oracle 서버의 n8n 워크플로우를 통해 메일·문자 전파를 자동화한다. Vercel 앱은 작업 입력, 공공데이터·AI 조합, 문서팩 편집, 전파 요청을 담당하고, n8n은 채널별 provider 연동과 발송 결과 회신을 담당한다. 제출 기준 채널은 메일과 문자이며, 카카오·밴드는 채널 승인 이후 확장 기능으로 둔다.

## n8n 표기 위치

- `AI 활용명`에는 n8n을 넣지 않는다. n8n은 AI가 아니라 workflow automation 도구다.
- `활용 AI명`은 `제미나이` 중심으로 입력한다.
- `서비스 설명`, `사업계획 내 구현 방식`, `추진체계`, `기술구성` 항목이 있으면 n8n을 `전파 자동화 엔진`으로 적는다.
- `데이터명` 칸에는 n8n을 넣지 않는다. 데이터가 아니라 자동화 인프라다.
- 발표에서는 “문서 생성 후 n8n으로 메일·문자 전파와 로그화를 연결한다”고 설명한다.

## 가장 안전한 접수 조합

접수 화면에서 데이터 입력 칸이 제한되어 있으면 아래 순서로 넣는다.

1. `한국고용정보원_사업주훈련 훈련과정`
2. `한국산업안전보건공단_안전보건자료 링크 서비스`
3. `한국산업안전보건공단_국내재해사례 게시판 정보 조회서비스`
4. `기상청_단기예보 조회서비스`
5. `법제처 국가법령정보 공유서비스`

고용노동 계열 가점 또는 필수 데이터 강조가 필요하면 1~3번을 우선 입력한다. 제품 신뢰성 설명까지 고려하면 4~5번을 추가한다.

## LLM Wiki 연결 설명

SafeGuard의 LLM Wiki는 `data/safety-knowledge/*.json`과 `knowledge/wiki/**`에 저장된 기초 안전지식 DB를 먼저 조회하고, `/api/knowledge/match`에서 작업 문장과 위험요인을 매칭한다. live API에서 새로 들어온 법령, KOSHA, 기상, 교육, 재해사례 원본은 `/api/knowledge/ingest`로 원본 이벤트 후보가 되고, `/api/knowledge/regenerate`가 Gemini 또는 OpenAI fallback으로 위키 초안을 재생성한다. 현재 화면에서는 `/knowledge`에서 확인할 수 있다.
