# SafeClaw 공모전 접수 입력값 정리

이 문서는 공모전 신청서의 `AI 활용 여부`, `활용 AI명`, `공공데이터 활용`, `국가중점 데이터 활용`, `전체 항목명(컬럼명)` 칸에 바로 옮겨 적기 위한 제출용 정리본입니다.

## 1. AI 활용 여부

| 신청서 항목 | 입력값 |
| --- | --- |
| AI 활용 여부 | `사용` |
| 활용 AI명 | `제미나이` |
| 복수 선택 가능 시 | `제미나이`, `챗GPT` |
| 기타 AI/도구 표기 | OpenAI API는 일부 요약·검증 fallback, n8n은 AI가 아닌 전파 자동화 워크플로우로 설명 |

### AI 활용 설명

SafeClaw는 현장 작업 설명, 공공데이터 기반 근거, 기상 신호, 안전보건 자료를 조합해 위험성평가표, 작업계획서, TBM일지, 안전보건교육 기록, 외국인 근로자 안내문을 생성합니다. AI는 공공데이터 원문을 대체하지 않고, 법령·KOSHA·기상·교육·재해사례 근거를 문서별 조치 문장과 다국어 안전 공지로 재구성하는 역할을 합니다.

### 서비스 설명

SafeClaw는 현장관리자가 오늘 작업 내용을 한 줄로 입력하면 고용24, KOSHA, 기상청, 법제처 등 공공데이터와 AI를 조합해 위험성평가표, 작업계획서, TBM일지, 안전보건교육 기록, 외국인 근로자 안내문, 메일·문자 전파 메시지를 생성하는 산업안전 문서팩 서비스입니다. 실시간 기상 위험, 법령 근거, 유사 재해사례, 후속 교육 정보를 문서별 근거로 연결해 작업 전 판단과 교육·전파를 한 흐름으로 지원합니다.

### 구현 방식 설명

SafeClaw는 Vercel 기반 웹앱에서 작업 입력, 공공데이터·AI 조합, 문서팩 편집, 다운로드, 이력 저장을 처리합니다. 메일·문자 전파는 n8n 워크플로우와 provider 연동으로 처리하며, 발송 결과는 전파 로그로 남깁니다. 제출 기준 채널은 메일과 문자이고, 카카오·밴드는 채널 승인 이후 확장 기능으로 둡니다.

## 2. 가점 및 필수 데이터 체크

### 국가중점 데이터 활용

| 신청서 항목 | 입력값 |
| --- | --- |
| 국가중점 데이터 활용 | `예` |
| 대표 데이터 | `법제처 법령정보데이터` 또는 `국가법령정보` |
| 기관명 | `법제처` |
| 데이터 성격 | `국가중점데이터` |
| 활용 내용 | 산업안전보건법령·판례·법령해석례를 현장 작업 조건과 매칭하여 위험성평가표, 작업계획서, TBM일지, 안전보건교육 기록의 근거로 제공합니다. |

보조로 언급할 수 있는 국가중점 데이터 후보는 `한국에너지기술연구원_실시간 홍반자외선 데이터 제공 서비스`입니다. 옥외 작업의 자외선·열노출 위험 신호 확장 데이터로 설명하되, 주 가점 데이터는 법제처 법령정보데이터로 잡는 것이 가장 안전합니다.

| 신청서 항목 | 입력값 |
| --- | --- |
| 주관기관·고용부 계열 공공데이터 활용 | `예` |
| 대표 데이터 | `한국고용정보원_사업주훈련 훈련과정`, `한국산업안전보건공단_안전보건자료 링크 서비스`, `한국산업안전보건공단_국내재해사례 게시판 정보 조회서비스`, `한국산업안전보건공단_안전보건법령 스마트검색` |

## 3. 데이터 입력 권장 순서

신청서의 데이터 입력 칸이 제한되어 있으면 아래 순서로 입력합니다.

1. `한국고용정보원_사업주훈련 훈련과정`
2. `한국산업안전보건공단_안전보건자료 링크 서비스`
3. `한국산업안전보건공단_국내재해사례 게시판 정보 조회서비스`
4. `한국산업안전보건공단_안전보건법령 스마트검색`
5. `기상청_생활기상지수 조회서비스(3.0)`
6. `법제처 국가법령정보 공유서비스`
7. `한국에너지기술연구원_실시간 홍반자외선 데이터 제공 서비스`

## 5. 신청서 칸별 최종 추천 입력

### 데이터 보유 기관 분류

```text
고용부 및 산하기관
```

첫 번째 데이터로 `한국고용정보원_사업주훈련 훈련과정` 또는 `한국산업안전보건공단_안전보건자료 링크 서비스`를 넣을 때 가장 자연스럽습니다.

### 기관명

```text
한국고용정보원
```

또는 KOSHA 데이터를 먼저 넣는 경우:

```text
한국산업안전보건공단
```

### 데이터 성격

```text
공공데이터포털
```

국가중점 데이터 칸에는 별도로:

```text
국가중점데이터
```

### 데이터명

가장 안전한 1순위:

```text
한국고용정보원_사업주훈련 훈련과정
```

제품 핵심과 가장 직접 연결되는 2순위:

```text
한국산업안전보건공단_안전보건자료 링크 서비스
```

국가중점 가점용:

```text
법령정보데이터
```

### 전체 항목명(컬럼명)

고용24를 1번 데이터로 넣는 경우:

```text
과정명, 훈련기관명, 훈련시작일, 훈련종료일, 훈련지역, 직종/NCS분류, 훈련비, 수강대상, 과정상태
```

KOSHA 안전보건자료 링크 서비스를 1번 데이터로 넣는 경우:

```text
ServiceKey, pageNo, numOfRows, 제작형태, 업종, 재해유형, 외국어 구분, 자료명, 자료 링크 URL, 첨부파일 경로, 첨부파일명
```

법제처 국가중점 데이터를 넣는 경우:

```text
현행법령, 행정규칙, 판례, 법령해석례, 별표·서식, 법령용어, 법령명, 법령ID, 시행일자, 공포일자, 소관부처, 조문번호, 조문제목, 조문내용, 개정이력
```

## 6. LLM Wiki 연결 설명

SafeClaw의 LLM Wiki는 `data/safety-knowledge/*.json`, `knowledge/wiki/**`, Supabase `safety_reference_items`에 저장된 기초 안전지식 DB를 먼저 조회하고, `/api/knowledge/match`와 `/api/safety-reference/search`에서 작업 문장과 위험요인을 매칭합니다. live API에서 새로 들어온 법령, KOSHA, 기상, 교육, 재해사례 원본은 지식 후보로 누적하고, Gemini 또는 OpenAI fallback으로 문서별 보완 제안을 생성합니다. 현재 화면에서는 `/knowledge`, `/evidence`, `/ops/api`에서 확인할 수 있습니다.
