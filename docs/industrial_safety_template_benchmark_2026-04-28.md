## 핵심 벤치마킹 결과

### 1. SafeGuard 기본 문서팩에 반드시 들어가야 할 엑셀형 구조

한국지역난방공사용 `발주자 및 시공자 안전서류 점검` 계열 엑셀은 SafeGuard가 참고하기 좋은 시트형 구조를 갖고 있습니다.

| 시트 축 | SafeGuard 반영 방향 |
|---|---|
| 점검결과 요약 | 문서팩 첫 장에 업체, 현장명, 점검일자, 공정률, 주요 결과 요약을 자동 생성 |
| 위험성평가의 적정성 | 위험성평가 결과를 단순 표가 아니라 실행 여부, 확인 근거, 개선 의견까지 연결 |
| 작업계획서 작성 상태 | 작업계획서 유무, 작업순서, 안전대책 반영 여부를 별도 탭으로 생성 |
| 시공자 산업재해 대응체계 | 비상연락, 보고체계, 응급조치, 사고 발생 시 조치 흐름을 문서팩에 포함 |
| 안전보건대장 작성관리 | 발주자/시공자 관점의 관리 문서 점검 항목으로 확장 |

현재 SafeGuard의 위험성평가/TBM/교육 기록 흐름에 이 구조를 붙이면, 단순 생성물이 아니라 `현장 점검 대응 가능한 안전 문서팩`처럼 보이게 됩니다.

### 2. 시공사 안전관리 수준 평가표에서 가져올 평가 축

`시공사 안전관리 수준 평가표` 계열 엑셀은 아래 시트 구성이 반복됩니다.

| 평가 축 | SafeGuard 적용 |
|---|---|
| 위험성평가 | 위험요인, 감소대책, 잔여위험, 담당자, 확인 근거 |
| 작업계획서 작성 및 이행절차 | 작업순서, 허가서, 장비/인원 투입, 작업 전 확인 |
| 위험작업 안전관리 | 굴착, 양중, 장비 이동, 고소, 화기 등 고위험 작업별 체크 |
| 시공사 안전보건관리 | 협력사 교육, 보호구, 안전표지, 작업중지권, 회의체 |
| 관리감독체계 | 현장소장, 관리감독자, 안전관리자, 근로자대표 확인 흐름 |
| 맞춤형 컨설팅 실시 | AI 추천/후속 교육/개선 권고를 별도 섹션으로 표기 |

이 구조는 SafeGuard의 `산출물 생성` 이후 단계인 `점검 대응/개선 권고` 기능으로 확장하기 좋습니다.

### 3. HWPX 절차서에서 가져올 문서팩 종류

`4.0 권고사항(절차서 및 서식)`의 HWPX 4종은 제품 템플릿 품질을 올리는 데 가장 직접적으로 유용합니다.

| 문서 | 반영할 제품 기능 |
|---|---|
| 공사감독일지 및 안전감독일지 서식 | 작업 전/후 일지, 안전관리 특이사항, 품질관리/기타사항 기록 |
| 중대재해 발생 대비 대응 절차서 | 비상 대응 절차, 보고체계, 역할 분담, 사고 발생 후 확인 항목 |
| 열수송관공사 표준 작업계획서 | 작업구간, 공사명, 작성자/확인자, 작업계획 확인 순서, 안전작업허가서 첨부 항목 |
| 열수송관공사 위험성평가 실시규정 | 현장소장, 안전관리자, 위험성평가 담당, 관리감독자, 근로자대표 역할 |

특히 표준 작업계획서에는 `작성 주체`, `확인 순서`, `안전작업허가서 첨부`, `현장여건에 맞춘 변형`이라는 현장형 문구가 있어 SafeGuard export의 안내문에 반영할 가치가 큽니다.

### 4. 이미지 자료에서 가져올 제품 경험

이미지 자료는 원본을 제품에 포함하지 않고, 제품 기능 설계에만 반영해야 합니다.

| 이미지 유형 | 제품 기능화 |
|---|---|
| 현장점검 사진 | 작업 전 사진/작업 후 사진 첨부 슬롯 |
| 강평 이미지 | 점검 후 개선 권고 요약 카드 |
| 작업중지권 게시 | TBM/교육 기록에 작업중지 기준 확인 체크 |
| 신호수 개선 사례 | 장비 동선/신호수 배치 개선 제안 |
| 위험성평가 샘플 이미지 묶음 | 산출물에 사진 근거 첨부 가능 구조 |

SafeGuard 화면에는 `사진 첨부` 기능을 바로 무겁게 구현하기보다, 먼저 문서팩 export에 `사진 첨부 위치`, `사진 설명`, `조치 전/후`, `확인자` 필드를 넣는 것이 현실적입니다.
