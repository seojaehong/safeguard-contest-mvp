# SafeGuard 최우수 도전 공백 리스트

## 한 줄 소개
오늘 작업 설명 한 줄을 위험성평가·TBM·안전교육 기록으로 바꾸는 산업안전 코파일럿

## 현재 판단
SafeGuard는 상위권 도전 가능 상태다. 다만 최우수권으로 올라가려면 아래 공백을 우선순위대로 메워야 한다.

## 우선순위 1
### KOSHA 축 보강
- 현재 상태: 공식 가이드 링크 기반 fallback 보강
- 왜 중요한가: 공식 안전기관 축이 약하면 데이터 활용성의 압도감이 떨어진다.
- 보강 방향: API 또는 데이터 경로를 더 붙이거나, 불가능하면 공식 자료 기반 판단 연결을 화면과 발표에서 더 명시한다.
- 발표 문장: KOSHA는 현재 공식 자료 기반 보강 축으로 사용 중이며, 다음 단계에서 더 강한 데이터 경로로 확장합니다.

## 우선순위 2
### 가점 데이터 확정
- 현재 상태: 고용24 교육 연계는 구현돼 있으나 가점 대상 확정 문구는 아직 보수적으로 써야 한다.
- 왜 중요한가: 최우수권에서는 가점 증빙과 기관 연계성이 강한 무기가 된다.
- 보강 방향: 데이터 출처, 기관, 가점 인정 가능 여부를 문서로 정리하고 모든 발표자료에 같은 문구로 넣는다.
- 발표 문장: 고용24 연계 데이터는 외국인 근로자와 신규 투입자 교육 연결에 사용하고 있습니다.

## 우선순위 3
### 발표 압축도 강화
- 현재 상태: 문서는 풍부하지만 첫 30초 압축 메시지는 더 날카롭게 만들 수 있다.
- 왜 중요한가: 최우수권은 기능 수보다 발표 이해 속도가 좌우한다.
- 보강 방향: 첫 30초에 문제, 제품, 산출물 가치가 끝나도록 스크립트와 슬라이드를 더 압축한다.
- 발표 문장: SafeGuard는 검색기가 아니라, 오늘 작업을 실행 문서로 바꾸는 도구입니다.

## 우선순위 4
### 증빙 시각화
- 현재 상태: 증빙 파일은 충분하지만 한 장 표로 압축한 심사위원용 시각 자료는 약하다.
- 왜 중요한가: 심사위원은 긴 로그보다 요약표를 선호한다.
- 보강 방향: 데이터 상태, 반영 위치, 증빙 파일 경로를 한 장 표로 정리한다.
- 발표 문장: 어떤 데이터가 어떤 산출물에 연결되는지 이 표 한 장으로 설명할 수 있습니다.
