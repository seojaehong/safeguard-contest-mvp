# SafeClaw 상용 SaaS v1 출시 후보 리포트

## 기준

SafeClaw v1은 완성형 엔터프라이즈 제품이 아니라, 현장 1~3곳에 유료 파일럿으로 제안 가능한 작업공간을 목표로 한다. 기준은 한 줄 작업 입력에서 문서팩 생성, 근거 확인, 작업자·교육 확인, 메일·문자 전파, 이력 재조회까지 끊기지 않는 흐름이다.

## 이번 라운드에서 닫은 항목

- 문서 품질 하네스가 허가서(`workPermitDraft`)를 별도 정식 문서로 검증한다.
- 20개 대표 현장 입력과 5개 변형 조건을 조합한 품질 매트릭스가 `evaluation/saas-v1/matrix-runner-report.json`에 생성된다.
- 위험성평가표, 작업계획서, 허가서, TBM 브리핑, TBM 일지, 안전보건교육 기록, 사진·증빙, 외국인 안내문, 현장 공유 메시지가 문서별 요구 키워드와 시나리오 위험요인을 통과 기준으로 점검된다.
- 하네스 기본 출력 위치를 `evaluation/saas-v1/`로 고정해 출시 후보 검증 산출물이 한 폴더에 모인다.

## 현재 제품 상태

| 축 | 현재 판정 | 근거 |
|---|---|---|
| 관리자 이력 | 제출 후보 | `/archive`, `/home`, `/workspace`에서 최근 문서팩과 전파 이력 경로를 제공한다. 실제 서버 이력은 Supabase 설정과 로그인 상태에 따라 활성화된다. |
| 문서 서식 | 제출 후보 | `WorkpackEditor`가 위험성평가표, 작업계획서, 허가서, TBM 일지별 HTML 표 구조와 HWPX 텍스트형 제출 구조를 분리한다. |
| 근거 연결 | 제출 후보 | 법령, KOSHA, 재해사례, 기상, 교육 추천은 문서 반영 근거와 보완 생성 흐름에 연결된다. 직접 근거와 보조 근거의 분리는 계속 유지해야 한다. |
| 다운로드 | 제출 후보 | XLS/HWPX/PDF 보조 제출 흐름은 제공된다. HWPX는 원본 셀 단위 복제가 아니라 제출형 구조로 고정한다. PDF는 브라우저 인쇄 기반이다. |
| 디자인 셸 | 제출 후보 | SafeClaw 브랜드 셸과 제품 메뉴는 적용됐다. 디자이너 최종 픽셀안은 별도 패스로 정렬한다. |
| 전파 | 제출 후보 | 메일·문자는 정식 채널, 카카오·밴드는 승인 전 잠금 상태로 운영한다. |

## 남은 공지성 리스크

- HWPX는 공공기관 원본 양식의 셀 병합과 페이지 배치를 그대로 복제하지 않는다. v1에서는 “제출 보조 초안”으로 안내한다.
- PDF는 서버 바이너리 렌더러가 아니라 브라우저 인쇄 흐름이다. 제출 전 사용자가 미리보기로 확인해야 한다.
- Supabase 저장 이력은 배포 환경 변수와 로그인 상태가 맞을 때 완전 동작한다. 비회원은 브라우저 임시 저장으로 안내한다.
- 카카오·밴드는 승인 완료 전까지 비활성 채널이다.

## 출시 후보 판정

판정: `pass_with_notice`

SafeClaw는 공모전 제출 및 유료 파일럿 제안에 필요한 핵심 루프를 갖췄다. 단, “법적 효력 있는 최종 문서”가 아니라 “공식자료 기반 현장 검토용 초안”으로 설명해야 하며, HWPX 원본 셀 복제와 서버 PDF 렌더링은 다음 버전의 고급 기능으로 분리한다.
