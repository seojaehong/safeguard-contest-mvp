# SafeGuard 시연 채널 결정 메모

## 결론
SafeGuard의 발표심사 시연 채널은 `웹 기반 메인 + 카카오톡 보조`로 결정하는 것이 가장 유리하다.

## 이유
- 웹은 문제 상황 입력부터 위험 요약, 즉시 조치, 위험성평가, TBM, 안전교육 기록, 근거 출처, 상태 표시까지 제품의 전체 가치사슬을 한 화면에서 보여줄 수 있다.
- 카카오톡은 별도 제품 채널이라기보다, 웹에서 생성된 결과를 작업반장과 작업자에게 전달하는 `현장 전파 수단`으로 설명할 때 가장 설득력이 높다.
- 발표심사는 제품 및 서비스 개발 부문 기준으로 시제품이 구체화되어야 하고, 발표심사 전까지 웹 URL을 업데이트할 수 있으므로 웹 중심 운영이 리스크가 적다.

## 공모전 기준 반영
- 공모전 FAQ 기준으로 제품 및 서비스 개발 부문은 접수 시점에 완성본이 아니어도 되지만, 2026년 7월 21일 발표심사 전까지 시제품이 구체화되어야 한다.
- FAQ 기준으로 발표심사 전까지 웹 URL은 계속 업데이트할 수 있다.
- 공모요강에는 제품 및 서비스 개발 부문 수상작에 대해 발표심사 전 `웹/앱 등 동영상 시연이 가능한 파일` 제출 의무가 명시되어 있다.

## 추천 시연 동선
1. 홈 웹 화면에서 대표 시나리오 입력
2. 위험 요약과 즉시 조치 확인
3. 위험성평가표, TBM 브리핑, TBM 일지, 안전교육 기록 초안 확인
4. 근거 출처와 live / partial / mock 상태 설명
5. 마지막에 카카오톡 전파용 메시지를 보여주며 실제 현장 전파성을 설명

## 발표용 메시지
- `SafeGuard의 메인 인터페이스는 웹입니다. 심사위원에게는 전체 산출물 체인을 웹에서 시연하고, 카카오톡은 현장 적용 단계에서 바로 전파되는 실행 채널로 보여드립니다.`
- `즉, 웹은 판단과 생성, 카카오톡은 전달과 실행을 담당합니다.`

## 지금 기준 최종 판단
- 메인 채널: 웹
- 보조 채널: 카카오톡
- 제외 판단: 카카오톡 단독 시연은 근거, 상태, 산출물 체인을 충분히 보여주기 어려워 메인 채널로는 부적합
