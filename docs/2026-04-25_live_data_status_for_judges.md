# SafeGuard 데이터 소스 상태 요약

심사 대응을 위해 현재 앱에서 사용 중인 데이터 소스와 상태를 정리했다. 핵심 원칙은 모든 경로를 `실시간`으로 과장하지 않고, live / partial / mock / planned를 구분해 설명하는 것이다.

| 데이터 소스 | 역할 | 현재 상태 | 데모 반영 여부 | 비고 |
| --- | --- | --- | --- | --- |
| Gemini API | 질문형 응답 생성, 실무 요약 문장 구성 | live | 예 | 대표 시나리오 기준 live 응답 검증 완료 |
| OpenAI API | 대체 AI 경로 | planned | 아니오 | 현재는 Gemini 우선 사용 |
| Law.go mock dataset | 법령/판례 근거 카드, 데모 검색 | mock | 예 | 홈 화면과 검색 보조 화면에 사용 |
| korean-law-mcp | 법령 근거 보강 | planned | 조건부 | 활성화 시 source mix로 표시 가능 |
| 공공데이터 키 | 향후 기상/안전 데이터 연계 | partial | 제한적 | 키는 확보했지만 앱 주경로 어댑터는 추가 구현 필요 |

## 왜 fallback이 필요한가
- 공공 API와 생성형 AI는 응답 지연, quota, timeout, 일시 장애가 발생할 수 있다.
- SafeGuard는 데모 중 화면이 멈추지 않도록 timeout 20초, 1회 retry, 실패 시 graceful fallback 정책을 사용한다.
- 이 정책은 기능 축소가 아니라, 현장형 도구에서 끊김 없는 전달을 우선한 설계다.

## 발표 시 권장 문구
- `SafeGuard는 live 경로를 우선 사용하되, 실패 시에도 위험성평가와 TBM 초안 흐름이 끊기지 않도록 fallback을 설계했습니다.`
- `현재는 AI 응답은 live 검증을 마쳤고, 일부 법령/공공데이터 경로는 mock 또는 partial 상태를 투명하게 공개하고 있습니다.`

## 외국인 근로자 교육 연계 메모
- 외국인 근로자 대상 취업·안전교육 정보 연계는 제품 스토리와 매우 잘 맞는다.
- 다만 공모전 가점 인정 여부는 해당 데이터 출처가 실제 가점 대상 기관 또는 데이터 범주인지 마지막 확인이 필요하다.
