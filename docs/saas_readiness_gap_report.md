# SafeClaw 상용 SaaS 준비 갭 리포트

## 현재 닫힌 축

- 작업 입력에서 문서팩 생성까지의 핵심 엔진은 동작한다.
- 법령, KOSHA, 기상, 교육 추천, 재해사례, 내부 지식 DB를 문서 근거로 연결하는 흐름이 있다.
- 문서 보완은 자동 덮어쓰기가 아니라 제안, 편집, 삽입 방식으로 설계되어 안전서류 신뢰를 해치지 않는다.
- 메일·문자 전파는 제출 기준 채널로 두고, 카카오·밴드는 승인 전 보류 상태로 분리했다.
- 서버 이력 화면은 관리자 세션이 있을 때 문서팩과 전파 로그를 조회하는 방향으로 정리됐다.
- 문서 편집 화면에 고객사 원본 서식 매핑 준비 흐름을 추가했다.

## 유료 SaaS까지 남은 핵심 갭

### 1. 사업장별 DB와 아카이브

상용 고객은 매일 새 문서를 만드는 것보다 어제의 문서, 작업자, 교육 이력, 전파 로그를 다시 열어 수정하고 재사용하기를 원한다. 따라서 다음 축이 필요하다.

- 현장별 문서팩 목록.
- 날짜별 작업 스냅샷.
- 작업자 명단 재사용.
- 교육 확인 이력.
- 전파 결과와 실패 사유 로그.
- 문서 재오픈 후 재생성·재다운로드.

### 2. 고객사 서식 렌더링

대기업과 공공기관은 자체 서식이 있다. SafeClaw 표준 서식만으로는 파일럿 설명은 가능하지만, 유료 전환에는 고객사 양식 반복 출력이 필요하다.

- XLSX/HWPX 원본 업로드.
- 필드 매핑.
- 검수 완료된 템플릿만 사용.
- 원본 셀 단위 복제는 별도 QA 게이트.

### 3. 출력 품질

현재 HWPX는 텍스트 기반 제출형이며, PDF는 브라우저 인쇄 흐름이다. 상용 기준에서는 다음이 필요하다.

- 서버 PDF 렌더러.
- HWPX 원본 템플릿 치환.
- Excel 인쇄영역, 페이지 나눔, 결재란 검수.
- 서식별 회귀 테스트.

### 4. 근거 정밀도

근거는 많이 붙는 것보다 현재 작업에 정확히 붙는 것이 중요하다.

- 법령·해석례·판례를 문서별 반영 위치로 연결.
- 직접 근거와 보조 근거 구분.
- KOSHA 자료와 재해사례는 작업유형 조건부로 노출.
- 사용자에게 “왜 이 근거가 붙었는지”를 짧게 보여준다.

### 5. 운영 보안과 권한

유료 SaaS는 안전서류와 연락처를 다루기 때문에 최소한의 운영 보안이 필요하다.

- 조직, 현장, 사용자 역할.
- RLS 정책 검증.
- 연락처 마스킹.
- 전파 동의와 목적 고지.
- 감사 로그.

## 1차 유료 파일럿 기준

- 고객사 1곳, 현장 1~3개.
- 표준 문서팩 + 고객사 주요 3종 서식 매핑.
- 문서 재오픈과 전파 로그 조회.
- 메일·문자 실발송.
- PDF/HWPX/XLS 수동 검수 통과.

## 아직 말하면 안 되는 표현

- 모든 원본 서식 완전 복제.
- 법적 효력 보장.
- 사고 책임 면제.
- 모든 공공기관 제출 통과.

## 말할 수 있는 표현

- 공식자료 기반 안전서류 초안.
- 현장 확인 후 사용하는 제출 보조 문서.
- 고객사 서식 매핑형 파일럿.
- 문서팩, 근거, 전파, 이력을 한 흐름으로 관리.
