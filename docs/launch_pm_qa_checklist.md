# SafeGuard D-1 PM QA Checklist

점검일: 2026-04-29

목적: 내일 런칭 전 회의에서 나온 화면/기능 리스크를 실제 QA 항목으로 고정한다. 범위는 작업 입력, API 조합, 문서팩 편집, 다운로드, 근로자·교육, 현장 전파, 근거 링크, 모바일 반응형이다.

## Launch Decisions

| 항목 | 결정 | 제품 반영 |
| --- | --- | --- |
| 관리자 계정 미연결 | GA 차단이 아니라 이력 저장만 로그인 후 활성화 | 상단은 `문서팩 준비됨`, 저장 패널은 `로그인 후 저장 가능`으로 분리 |
| 첫 진입 시드 데이터 | 첫 방문자는 최근 문서가 있는 것처럼 보이지 않게 처리 | 생성 전에는 빈 상태, 생성 후 현재 문서팩만 최근 이력처럼 표시 |
| 전파 채널 | 메일·문자만 1차 정식, 카카오·밴드는 준비 중 | 카카오·밴드 disabled + 준비 중 배지 |
| 다운로드 형식 | PDF/XLS/HWPX를 1차 정식, 나머지는 베타 | DOC/TXT/JSON/CSV/HTML/JPG는 `베타 형식` 아래로 이동 |
| 근거 링크 | 작성 중인 문서 화면을 잃지 않게 새 탭 | Law.go/KOSHA 근거 링크 새 탭 처리 |

## Functional QA

| 영역 | 체크 항목 | 기대 결과 | 상태 |
| --- | --- | --- | --- |
| 작업 입력 | 6개 현장 예시 선택 | textarea와 현장 브리프가 즉시 바뀌고 자동 생성은 실행되지 않음 | 대기 |
| 작업 입력 | 599/600/601자 입력 | 경계 상태에서 카운터와 제출 동작이 안정적임 | 대기 |
| API 선조회 | 지역·날씨가 있는 입력 | 생성 전 기상청 현재 조회가 현장 브리프에 반영됨 | 대기 |
| API 조합 | 문서팩 생성 | 기상청, Law.go, Gemini, Work24, KOSHA 자료, KOSHA 교육, 재해사례 호출 결과가 문서 반영 근거에 남음 | 대기 |
| 단계 이동 | 6단계 상단 탭 클릭 | 각 고유 섹션으로 이동하고 입력/편집 값이 유지됨 | 대기 |
| 근거 | 문서 반영 근거 빈 응답 | 빈 카드가 아니라 명확한 empty state가 표시됨 | 반영 |
| 근거 | Law.go/KOSHA 링크 클릭 | 새 탭으로 열리고 작성 중 화면은 유지됨 | 반영 |
| 문서팩 | 11종 문서 탭 전환 | 각 문서 내용이 독립적으로 표시되고 편집 가능 | 대기 |
| 문서팩 | 편집 후 새로고침 | 같은 브라우저에서 로컬 자동저장 값이 복원됨 | 반영 |
| 다운로드 | PDF/XLS/HWPX | 파일 생성 또는 인쇄 흐름이 정상 동작하고 한글이 깨지지 않음 | 대기 |
| 다운로드 | 베타 형식 | DOC/TXT/JSON/CSV/HTML/JPG가 숨김 영역에서만 노출됨 | 반영 |
| Google Sheets | 시트 열기 | 새 시트가 열리고 TSV 복사 또는 TSV fallback이 동작 | 대기 |
| 근로자 | 빠른 추가 | 동의 체크 전에는 추가 불가, 체크 후 명단과 전파 대상에 반영 | 대기 |
| 다국어 | 외국인 언어 선택 | 메시지 미리보기와 전파 payload의 언어가 일치 | 대기 |
| 전파 | 메일·문자 전송 | 전송 전 확인 패널이 뜨고 provider 결과가 채널별로 표시 | 대기 |
| 전파 | 카카오·밴드 클릭 | 클릭 불가 또는 준비 중 안내로 오해가 없음 | 반영 |
| 이력 | 로그인 전 저장 클릭 | 로그인 후 저장 가능하다는 안내만 표시 | 대기 |

## Device QA

| --- | --- | --- | --- |

## Release Notes

- DB migration은 이번 QA 패치 범위에서 실행하지 않는다.
- 카카오·밴드 실제 발송은 승인 후 활성화한다.
- SMS/메일 provider는 실제 발송 테스트 결과를 n8n 실행 ID와 함께 별도 기록한다.
