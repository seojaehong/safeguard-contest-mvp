# 법령 전문·AI 근거 매핑 운영 기준

## 목적

SafeGuard의 법령·해석례·판례 근거는 공식 API 결과를 원천으로 삼고, Gemini는 근거를 새로 만들지 않고 `선별`, `정렬`, `연결 이유 작성`, `문서 반영 위치 설명`에만 사용한다.

## 법령 전문 렌더링

- Law.go 상세 API가 조문 단위를 반환하면 화면에는 `[법령 전문]`으로 전체 조문을 표시한다.
- 기존처럼 안전 관련 키워드가 있는 조문 일부만 잘라 보여주지 않는다.
- Law.go 상세 호출이 실패하면 캐시된 기초 조문 스냅샷을 사용하되, 화면과 문서에는 `기초 조문 스냅샷`임을 명확히 표시한다.
- 장기 운영에서는 산업안전보건법, 시행령, 시행규칙, 중대재해처벌법, 시행령을 증분 저장 대상으로 두고, 공포번호·시행일자 변경이 있을 때만 다시 파싱한다.

## Gemini 근거 매핑 계층

Gemini는 다음 단계에서만 개입한다.

1. 검색된 법령·해석례·판례 후보 목록을 작업 조건과 비교한다.
2. 법령은 1차 근거, 해석례는 적용범위 보조 근거, 판례는 이행 여부 점검 보조 근거로 정렬한다.
3. 각 근거의 `summary`를 “어떤 문서 문장에 왜 반영되는지” 중심으로 다시 쓴다.
4. 재정렬된 근거는 화면 카드, 위험성평가표, TBM 기록, 안전보건교육 기록, 사진/증빙 문서에 동일하게 반영한다.

## 금지 원칙

- Gemini가 존재하지 않는 조문, 판례, 해석례, 사건번호, 회신일자를 생성하면 안 된다.
- 공식 원문을 대체하는 최종 법률 판단처럼 표현하지 않는다.
- 해석례와 판례는 현장 조치의 보조 검토 근거로만 표현한다.

## 제출 전 확인 항목

- 법령 상세 페이지에서 Law.go 상세 호출 성공 시 `[법령 전문]`이 보인다.
- 해석례 카드는 “현재 작업 조건과 유사한 위험 판단 기준” 같은 일반 문구가 아니라 작업 조건과 문서 반영 위치를 포함한다.
- 위험성평가표, TBM 기록, 안전보건교육 기록의 `[법령·해석례 반영]` 섹션이 화면 근거 카드와 같은 연결 이유를 사용한다.
- AI 근거 매핑 실패 시 원래 공식 API 검색 결과 순서로 돌아가고, 서버 로그에 실패 사유가 남는다.
