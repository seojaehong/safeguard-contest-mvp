# SafeGuard 현재 루브릭 평가 및 유료화 가능성

## 결론

SafeGuard는 현재 `제출 가능`을 넘어 `수상권 진입 가능` 상태입니다. 다만 최우수권을 안정적으로 노리려면 발표와 증빙에서 “데모가 잘 된다”가 아니라 “현장 업무가 닫힌 루프로 돌아간다”를 더 강하게 보여줘야 합니다.

유료화는 `파일럿/컨설팅 결합형`으로는 지금도 시도 가능하지만, `셀프서브 SaaS`로 바로 과금하기에는 아직 부족합니다. 이유는 명확합니다. 계정, 사업장별 데이터 저장, 권한, 발송 이력, 법적 책임 문구, 고객별 채널 설정, 운영 모니터링이 아직 제품 내부에 닫혀 있지 않습니다.

## 현재 수준 판단

| 항목 | 현재 판단 | 이유 | 다음 보강 |
|---|---|---|---|
| 완성도 | 강함 | 홈에서 작업 입력, 문서팩 생성, 근거 확인, 다운로드, n8n 전파까지 이어짐 | 제출 전 대표 시나리오 고정 리허설 |
| 데이터·AI 활용성 | 강함 | Law.go, Gemini, 기상청, Work24, KOSHA 교육, KOSHA 공식자료, KOSHA 재해사례가 연결됨 | 데이터가 어느 문서 문장에 반영됐는지 화면에서 더 선명하게 표시 |
| 실용성 | 강함 | 위험성평가, 작업계획서, TBM, 교육기록, 비상대응, 증빙까지 현장 문서 흐름을 만듦 | 사업장별 저장과 반복 작업 재사용 |
| 차별성 | 강함 | 단순 검색이 아니라 실행 문서팩과 전파까지 연결됨 | 경쟁 서비스 대비 “당일 입력 즉시 문서팩” 메시지 고정 |
| 확산가능성 | 보통~강함 | 외국인 근로자 브리핑, 후속 교육, 다업종 예시가 있음 | 관리자/근로자/외국인 근로자별 UX 분리 |
| 신뢰성 | 보통~강함 | API 연결 감사와 n8n 발송 증빙이 있음 | 운영 대시보드, 장애 알림, API 실패 이력 저장 |

## 수상 가능성 판단

현재 상태는 `본선 제출 경쟁력 높음`, `수상권 진입 가능`, `최우수권은 발표 완성도와 증빙 설계에 좌우`로 판단합니다.

강점은 세 가지입니다.
- 문제정의가 분명합니다. 현장은 위험 판단보다 기록, 공유, 교육, 증빙을 동시에 요구받습니다.
- 데이터 활용 스토리가 좋습니다. 법령, 기상, 교육, 재해사례, 공식 KOSHA 자료가 문서팩에 연결됩니다.
- 시연 흐름이 좋습니다. 한 줄 입력 후 위험성평가, 작업계획서, TBM, 교육기록, 전파까지 이어집니다.

약점도 분명합니다.
- 아직 “저장된 업무 시스템”보다는 “강한 생성형 워크스페이스”에 가깝습니다.
- 근거가 문서 어디에 반영됐는지 심사위원이 즉시 읽기에는 더 압축이 필요합니다.
- 카카오/밴드 같은 채널은 실제 운영까지 가려면 템플릿 승인과 채널 credential 절차가 남아 있습니다.
- 사용자 계정, 사업장 관리, 이력 저장이 없어 반복 사용 제품처럼 보이는 힘은 아직 약합니다.

## 유료화 가능성

### 지금 가능한 유료화

지금 당장 가능한 형태는 `현장 컨설팅/공동안전관리/소규모 사업장 파일럿`입니다.

판매 문장은 이렇게 잡는 것이 현실적입니다.

> 작업 전 입력 한 번으로 위험성평가, 작업계획서, TBM, 안전보건교육 기록, 외국인 근로자 안내문, 현장 전파 메시지를 생성하고 보관하는 안전 문서팩 자동화 도구

이 형태에서는 월 구독보다 `도입 파일럿`, `현장별 세팅`, `문서 서식 커스터마이징`, `n8n 전파 채널 연동`을 묶는 방식이 더 현실적입니다.

### 아직 어려운 유료화

바로 셀프서브 SaaS로 받기 어려운 이유는 아래와 같습니다.

- 사업장별 계정, 권한, 작업 이력 저장이 없습니다.
- 문서 생성 결과의 법적 책임 범위와 검토 플로우가 제품에 내장돼 있지 않습니다.
- 고객별 발송 채널 credential을 안전하게 등록·검증하는 UI가 없습니다.
- 작업자 서명, 교육 참석 확인, 사진 증빙 업로드, 변경 이력 추적이 아직 부족합니다.
- 관리자 화면에서 API 상태, 발송 실패, 재시도, 누락 문서 확인이 어렵습니다.

## 경쟁 구도 참고

경쟁 서비스는 이미 위험성평가, TBM, 교육, 점검, 사고/아차사고, 외국어 지원, 모바일 공유를 강조하고 있습니다. 예를 들어 스마트위험성평가V2는 위험성평가, TBM, 안전교육, 일일점검, 작업중지권, 아차사고/재해사례, 다국어 지원을 기능으로 제시합니다. 카스웍스는 카카오톡 기반 TBM 접근과 서명·관리 편의성을 강조합니다. 안전함은 위험성평가/TBM 공유와 외국인 근로자 자동 통역 공유를 강조합니다.

따라서 SafeGuard는 “우리도 TBM 됩니다”가 아니라 “공공데이터와 AI로 오늘 작업 문서팩을 즉시 만들고, 공식 근거와 전파까지 연결합니다”로 가야 합니다.

## 부족한 부분과 개선 계획

### 제출 전 마감

- 대표 시나리오 한 개를 고정하고, 발표 자료와 화면 문구를 완전히 맞춥니다.
- API 연결 감사, n8n 발송 증빙, Vercel 스모크 결과를 발표자료에 한 장으로 넣습니다.
- 근거 출처 카드에서 `반영 문서`와 `반영 문장`을 더 짧게 보여줍니다.
- 현장 전파 결과는 채널별 성공/건너뜀/실패를 분리해 보여줍니다.

### 파일럿 유료화

- 사업장 프로필, 작업자 명단, 기본 수신자, 언어 설정을 저장합니다.
- 문서팩 생성 이력과 다운로드 이력을 남깁니다.
- 작업자 확인 서명, 사진 증빙 업로드, 교육 참석자 기록을 추가합니다.
- 고객별 n8n credential 또는 발송 채널 설정 화면을 둡니다.

### SaaS 전환

- Supabase 기반 테넌트, 사용자, 사업장, 작업, 문서팩, 전파 로그 테이블을 설계합니다.
- 관리자/현장소장/근로자 권한을 분리합니다.
- 전파 실패 재시도와 장애 알림을 추가합니다.
- 문서 템플릿을 고객별로 선택·버전 관리합니다.
- 결제는 셀프서브보다 파일럿 고객 검증 이후 붙입니다.

## 우선순위

1. 현장 전파 결과 UI 개선
2. 근거-문서 반영 위치 시각화
3. 대표 시나리오 리허설과 제출 자료 정리
4. 사업장 프로필과 문서팩 이력 저장 설계
5. 작업자 확인/서명/사진 증빙 업로드
6. 고객별 채널 credential 관리
