# SafeGuard Product Workspace Upgrade

## 목적

SafeGuard를 제출용 원페이지 리포트가 아니라 현장 안전 작업공간으로 전환한다. 핵심 흐름은 `작업 생성 -> 근로자 배치 -> 문서팩 편집 -> 근거 확인 -> 현장 전파 -> 이력 저장`이다.

## 반영 내용

- 홈 화면 하단을 `FieldOperationsWorkspace`로 재편해 문서팩, 근거, 근로자, 교육, 전파, 저장을 한 작업공간에서 처리한다.
- 근로자 정보는 업무 적합성, 교육 확인, 현장 전파에 필요한 최소 정보로 제한했다.
- 연락처는 전파 자동화를 위해 포함하되, 화면에서는 마스킹 또는 전파 가능 상태 중심으로 보여준다.
- 국적과 주 사용 언어를 근로자 카드에 두어 외국인 근로자 안내문과 전파 메시지 선택에 연결한다.
- Supabase Auth 기반 관리자 로그인 패널과 저장 API를 추가했다.
- DB migration 파일은 추가했지만 실제 DB에는 적용하지 않았다.
- 문서팩 저장 시 근로자 명단, 작업자 배치 요약, 교육 확인 이력, 근거 요약이 함께 저장되도록 API를 분리했다.
- n8n 전파 이후 채널별 결과를 `dispatch_logs`로 저장할 수 있게 했다.

## 저장 구조

- `organizations`: 관리자 소유 사업장 단위
- `sites`: 현장 단위
- `workers`: 표시명, 역할, 현장 투입일, 경력/숙련도, 국적, 언어, 교육상태, 연락처
- `workpacks`: 질문, 시나리오, 산출물, 근거 요약, 작업자 배치 요약
- `education_records`: 작업별 교육 주제, 대상 근로자, 언어, 확인 상태
- `dispatch_logs`: 채널, 대상, 언어, provider 결과, 실행 ID

## 개인정보 최소화 기준

저장 대상은 업무 배치, 안전교육, 현장 전파에 필요한 정보로 제한한다. 주민번호, 여권번호, 체류자격, 주소, 생년월일, 가족정보는 저장하지 않는다.

## 제출·유료화 관점

이번 변경으로 SafeGuard는 단순 생성 화면이 아니라 현장 운영 이력이 남는 파일럿 제품 구조가 된다. 결제 기능은 붙이지 않고, 파일럿 도입과 현장 서식 커스터마이징, 전파 채널 연동을 1차 유료화 경로로 둔다.
