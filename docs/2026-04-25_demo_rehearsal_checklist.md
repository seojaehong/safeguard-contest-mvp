# SafeGuard Demo Rehearsal Checklist

## Goal

공모전 심사 기준에서 SafeGuard를 "검색기"가 아니라 "현장 산출물 생성형 안전 코파일럿"으로 보이게 만드는 최종 리허설 체크리스트입니다.

## Golden Path

1. 홈 `/`에서 대표 시나리오를 그대로 실행한다.
2. 오늘 위험 3개와 즉시 조치를 먼저 설명한다.
3. 위험성평가 초안, TBM 브리핑, TBM 일지 초안이 한 번에 생성되는 흐름을 보여준다.
4. 근거 출처와 신뢰성 상태를 보여주며 Law.go / korean-law-mcp / AI 상태를 설명한다.
5. 필요 시 `/ask`, `/search`, `/dryrun`은 보조 화면으로만 짧게 보여준다.

## Must-Check

- 홈 첫 화면만으로 제품 목적이 이해되는가
- 기본 시나리오 문장이 심사위원에게 직관적인가
- 위험 요약 → 즉시 조치 → 산출물 → 근거 순서가 자연스러운가
- `mock`, `live`, `fallback` 상태가 헷갈리지 않는가
- 드라이런 카드가 “검증된 데모”라는 인상을 주는가
- 검색/질문 화면이 메인 흐름을 방해하지 않고 보완하는가

## Demo Script Notes

- 첫 문장은 "오늘 작업 설명 한 줄을 위험성평가와 TBM 초안으로 바꾸는 안전 코파일럿"으로 통일한다.
- 기능 나열보다 "오늘 위험을 바로 행동과 문서로 번역한다"는 메시지를 반복한다.
- 실 API 실패 가능성은 숨기지 말고, fallback과 검증 로그로 신뢰성 대응을 설명한다.
- 다국어/TTS는 현재 기능이 아니라 향후 확장으로만 짧게 언급한다.

## Validation Checks

- `npm.cmd run typecheck`
- `npm.cmd run build`
- 홈 `/`에서 대표 시나리오 실행
- `/dryrun` 최신 로그 노출 확인
- citations 0건 또는 일부만 들어와도 레이아웃이 깨지지 않는지 확인
