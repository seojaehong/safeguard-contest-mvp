# SafeGuard KOSHA 축 및 가점 데이터 정렬 메모

## 한 줄 소개
오늘 작업 설명 한 줄을 위험성평가·TBM·안전교육 기록으로 바꾸는 산업안전 코파일럿

## KOSHA 축 현재 정의
- KOSHA는 현재 별도 API 어댑터가 아니라 `공식 Guide·교육자료 기반 보강 축`으로 사용한다.
- 즉, SafeGuard는 Law.go가 법령 근거 축이라면, KOSHA는 위험성평가와 TBM 문구를 공식 안전 실무자료로 보강하는 축으로 설명한다.
- 발표에서는 `KOSHA 공식 자료를 시나리오별로 매핑해 위험성평가와 TBM 문구를 더 실무적으로 보강했다`고 표현한다.

## KOSHA 발표 권장 문구
- SafeGuard는 Law.go로 법령 근거를 확보하고, KOSHA 공식 Guide와 교육자료로 위험성평가와 TBM 문구를 실무형으로 보강합니다.
- 현재 KOSHA는 공식 자료 기반 보강 경로이며, 다음 단계에서 더 강한 데이터 연계를 확장할 예정입니다.

## 가점 데이터 현재 정의
- 고용24 교육 연계 데이터는 외국인 근로자, 신규 투입자, 사업주 후속 교육 연결에 실제 사용 중이다.
- FAQ 기준으로 주요 기능이 아니라 보조 기능이어도 국가중점데이터 활용은 가점 검토 대상이 될 수 있다.
- 따라서 현재 문구는 `가점 후보 데이터` 또는 `가점 가능성이 높은 연계 데이터`로 통일한다.

## 가점 발표 권장 문구
- SafeGuard는 고용24 연계 데이터를 활용해 위험 알림에서 끝나지 않고 외국인 근로자와 신규 투입자를 위한 교육 연계까지 이어집니다.
- 이 데이터는 제품 핵심 흐름의 보조 축으로 동작하며, 공공데이터 활용성과 확장성을 동시에 보여줍니다.

## 문서상 금지 문구

- `KOSHA API live 완료`
- `모든 공공데이터가 실시간`
