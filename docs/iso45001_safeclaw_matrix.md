# SafeClaw ISO 45001식 안전보건 운영 매트릭스

## 목적

SafeClaw의 목표 수준은 `문서 자동 생성`이 아니라 작업 단위의 안전보건 운영 흐름을 닫는 것입니다. 이 문서는 SafeClaw 문서팩이 ISO 45001식 안전보건경영시스템의 주요 관점과 어떻게 맞물리는지 설명합니다.

이 문서는 인증 취득, 법적 효력, 공식 심사 통과를 보장하지 않습니다. 표현은 항상 `공식자료 기반 초안`, `현장 검토 후 제출 가능한 문서팩`, `ISO 45001식 문서체계에 맞춘 운영 보조`로 고정합니다.

## 운영 정의

| 운영 관점 | SafeClaw에서의 의미 | 대표 산출물 |
|---|---|---|
| Plan | 작업 전 위험을 식별하고 법규·공식자료 근거를 연결합니다. | 위험성평가표, 작업계획서, 근거 목록 |
| Do | TBM, 안전보건교육, 외국인 전송본으로 현장 실행을 시작합니다. | TBM 브리핑, TBM 기록, 안전보건교육 기록, 외국인 전송본 |
| Check | 조치 이행, 교육 확인, 사진·증빙, 전파 결과를 확인합니다. | 점검결과 요약, 사진/증빙, 교육기록, 전파 로그 |
| Act | 미조치 위험과 부적합을 후속조치·재발방지로 연결합니다. | 비상대응 절차, 개선조치 관리, 다음 작업 반영 |

## ISO 45001식 요구사항 매핑

| ISO 관점 | 요구 의미 | SafeClaw 기능 | 현재 상태 | 다음 보강 |
|---|---|---|---|---|
| 법규 요구사항 | 적용 법령과 공식자료를 식별하고 문서 문장에 연결 | Law.go, KOSHA, 고용노동부 자료를 근거 카드와 문서 하단에 반영 | 루브릭 `법규 요구사항 등록부 연결` 추가 | 법령 전문 캐시와 변경분 업데이트 이력 연결 |
| 위험성평가 | 작업활동별 유해·위험요인, 위험성 결정, 감소대책 수립 | 4M 체크형 위험성평가표와 즉시조치 생성 | 구현됨 | 위험요인별 법령·KOSHA 근거 자동 보완 제안 |
| 운영 통제 | TBM, 작업중지 기준, 보호구, 출입통제, 신호수 배치 | TBM 브리핑과 작업계획서에 실행 항목 생성 | 구현됨 | 현장 체크 완료 시각과 확인자 저장 |
| 역량·인식 | 작업자 교육, 신규·외국인 근로자 이해 확인 | 안전보건교육 기록, 쉬운 한국어, 다국어 전송본 | 구현됨 | 언어별 전송 결과와 회신 확인 저장 |
| 의사소통 | 관리자, 작업자, 외국인 근로자에게 필요한 내용 전파 | 메일·문자 전파, 현장 공유 메시지 | 메일·문자 중심으로 운영화 | 카카오·밴드 승인 후 보조 채널 연결 |
| 문서화 정보 | 문서번호, 작성자, 검토자, 승인자, 개정 이력, 보관 위치 | Excel/HWPX 서식과 루브릭 `문서관리와 개정 이력` | 기준 추가 | Supabase 이력 저장과 개정번호 자동 부여 |
| 성과평가 | 교육 확인, 조치 이행, 사진·증빙, 전파 로그 확인 | 점검결과 요약, 사진/증빙, dispatch logs | 일부 구현 | 사진 업로드와 증빙 파일 로그 연결 |
| 개선 | 미조치 위험, 부적합, 담당자, 기한, 조치결과 관리 | 루브릭 `부적합·개선조치 관리`와 비상대응 절차 | 기준 추가 | CAPA형 개선조치 레코드와 다음 작업 반영 |

## 11종 문서 재분류

| 등급 | 문서 | 역할 |
|---|---|---|
| 핵심 운영 문서 | 위험성평가표, 작업계획서, TBM 브리핑, TBM 기록, 안전보건교육 기록 | Plan과 Do를 실제 현장 실행으로 연결 |
| 확인 문서 | 점검결과 요약, 사진/증빙, 비상대응 절차 | Check와 Act의 증빙·후속조치 기반 |
| 전파 문서 | 외국인 근로자 출력본, 외국인 근로자 전송본, 현장 공유 메시지 | 작업자 이해와 현장 전파 |

## 제출 기준 품질 게이트

| 게이트 | 통과 조건 | 제품 표시 |
|---|---|---|
| 필수 확인 | 위험요인, 감소대책, 작업중지 기준, 교육 확인이 문서에 포함 | `충족` 또는 `보완 필요` |
| 제출 품질 | 확인자, 서명란, 보고체계, 증빙 위치가 포함 | `제출 품질 보강` |
| 현장 운영 | 외국인·신규 근로자 전달과 작업자 확인이 가능 | `현장 운영 추천` |
| ISO 운영체계 | 법규 요구사항, 문서관리, PDCA, 감사추적, 개선조치가 보임 | `ISO 운영체계` |

## 구현 반영 현황

- `lib/safety-document-rubric.ts`에 ISO 운영체계 분류와 5개 항목을 추가했습니다.
- `docs/public_safety_document_rubric.md`에 공공기관형 안전서류 루브릭과 ISO 45001식 해석을 반영했습니다.
- 작업공간 파일럿 체크리스트를 `PLAN / DO / CHECK / ACT / ISO` 구조로 바꿔 제품 화면에서도 운영체계 관점이 보이게 했습니다.
- Excel 일괄 export와 문서팩 편집 패널은 기존 루브릭 평가 결과를 그대로 사용하므로 새 ISO 운영체계 항목을 함께 노출합니다.

## 남은 보강

| 항목 | 필요 이유 | 제출 후 구현 방향 |
|---|---|---|
| 법령 전문 캐시 | API 응답 지연과 전문 누락을 줄이기 위함 | 법제처 전문을 증분 저장하고 변경분만 업데이트 |
| KOSHA 자료 위키화 | 공식자료 링크와 문서 반영 위치를 더 촘촘히 연결 | KOSHA Guide, 재해사례, 교육자료를 문서 항목별로 색인 |
| 문서 개정 이력 | 공공기관·대기업 제출 서식에서 문서관리 신뢰를 높임 | 문서번호, 개정번호, 작성·검토·승인 로그 저장 |
| 사진·증빙 업로드 | 점검 결과와 실제 현장 증빙을 연결 | 파일 로그, 촬영시각, 보관 위치, 문서 반영 위치 저장 |
| 개선조치 레코드 | 미조치 항목이 다음 작업으로 이어지게 함 | 담당자, 기한, 완료 확인, 재발방지 항목 저장 |

## 표현 금지선

- `ISO 인증 대체`라고 말하지 않습니다.
- `법적 효력 보장`이라고 말하지 않습니다.
- `관공서 제출 100% 통과`처럼 보장형 문구를 쓰지 않습니다.
- `실무자가 현장 검토 후 제출 가능한 초안`, `공식자료 기반 보조자료`, `안전보건 운영체계에 맞춘 문서팩`으로 설명합니다.
