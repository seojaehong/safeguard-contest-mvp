# SafeClaw 보안·개인정보 준비도

## 원칙

SafeClaw v1은 안전관리 업무에 필요한 최소 정보만 다룬다. 주민번호, 여권번호, 주소, 생년월일, 체류자격, 가족정보는 저장 대상이 아니다. 연락처와 이메일은 현장 전파와 교육 확인 목적에 한해 사용한다.

## 점검 결과

| 항목 | 상태 | 조치 |
|---|---|---|
| 환경변수 | 관리 필요 | `.env`와 토큰 파일은 커밋하지 않는다. Vercel과 로컬 환경변수로만 주입한다. |
| Supabase anon key | 공개 가능 범위 | 클라이언트에는 anon key만 사용한다. service role key는 서버 API와 운영 스크립트에서만 사용한다. |
| RLS | 확인 필요 | 운영 DB에서는 `organizations`, `sites`, `workers`, `workpacks`, `education_records`, `dispatch_logs`별 RLS 정책을 확인한다. |
| 연락처 표시 | 적용 필요 | 기본 화면에서는 마스킹하고, 전파 직전 확인 모달에서만 대상자를 명확히 보여준다. |
| 동의 문구 | 적용됨 | 작업자 추가 시 연락처·국적·언어 정보를 교육 확인과 현장 전파 목적으로 사용한다는 문구를 둔다. |
| 전파 로그 | 적용 필요 | 채널, 대상, 언어, provider status, 실행 ID, 실패 사유를 `dispatch_logs`에 남긴다. |
| 카카오·밴드 | 잠금 | 승인 전에는 선택 불가 상태로 유지한다. |

## UI 문구 기준

- 비회원: “비회원 임시 저장”
- 로그인 필요: “관리자 로그인 후 이력 저장”
- 저장 성공: “관리자 이력 저장 완료”
- 저장 실패: “저장 실패. 네트워크 또는 권한을 확인하세요.”
- 전파 실패: “발송 요청 실패”와 “통신사 전달 실패”를 구분한다.

## 운영 전 확인

- Supabase RLS 정책을 배포 DB에서 확인한다.
- Vercel Production에 `N8N_PUBLIC_BASE`, `N8N_WEBHOOK_PATH`, `N8N_WEBHOOK_TOKEN`만 넣고 `N8N_INTERNAL_BASE`는 넣지 않는다.
- 메일·문자는 제출 기준 정식 채널로 유지한다.
- 카카오·밴드는 승인 전까지 잠금 상태로 둔다.
