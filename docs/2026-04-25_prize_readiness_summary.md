# SafeGuard 수상권 점검 요약

## 한 줄 소개
오늘 작업 설명 한 줄을 위험성평가·TBM·안전교육 기록으로 바꾸는 산업안전 코파일럿

## 지금 강한 점
- 현장 설명 입력 한 번으로 위험 요약, 즉시 조치, 위험성평가 초안, TBM 브리핑, TBM 일지, 안전교육 기록 초안을 한 번에 보여준다.
- 공공데이터와 법령 근거를 활용해 단순 질의응답이 아니라 실무 산출물 생성 흐름을 제시한다.
- Gemini live 응답과 fallback 정책을 함께 갖춰 데모 안정성과 확장성을 동시에 설명할 수 있다.

## 지금 보완한 핵심
- 기존 문서에만 강하게 있던 `TBM → 위험성평가 → 안전교육` 연결을 홈 화면과 산출물에 실제로 반영했다.
- `오늘 팀에 다시 물어야 할 질문 3개`를 노출해 문서 자동작성보다 현장 대화 유도에 초점을 맞췄다.
- 외국인 근로자용 쉬운 문장 브리핑, 다국어, 교육기관 정보 연계는 본선 이후 확장 항목으로 정리했다.

## 심사 포인트
- 문제정의: 추락, 충돌, 강풍 같은 현장 고위험 작업을 즉시 문서화한다.
- 실무 적합성: 작업 전 회의와 교육기록까지 이어져 현장 운영의 닫힌 루프를 보여준다.
- 확장성: 외국인 근로자 브리핑, 교육 연계, 다국어/TTS로 확장 가능하다.
- 신뢰성: live, partial, mock, fallback 상태를 숨기지 않고 설명한다.

## 냉정한 리스크
- 법령/공공데이터 주경로는 여전히 mock 비중이 있어 `모든 데이터가 실시간이다`라는 인상은 피해야 한다.
- 외국인 근로자 교육 연계는 제품 메시지에는 매우 유효하지만, 공모전 가점 인정 대상 기관/데이터인지 최종 확인이 필요하다.
- 발표에서는 검색 기능보다 현장 산출물 생성과 실행 지원을 전면에 두는 편이 유리하다.

## 남은 4-7일 최우선 체크
1. 홈 데모에서 위험성평가, TBM, TBM 일지, 안전교육 기록이 같은 시나리오로 자연스럽게 이어지는지 최종 확인
2. live, partial, mock 데이터 사용 범위를 발표 문구와 동일하게 맞추기
3. 외국인 근로자·교육 연계 데이터가 가점 대상 기관/데이터인지 마지막 확인
4. 발표와 제출 문서에서 `검색 서비스`보다 `현장 산출물 생성형 안전 코파일럿` 메시지로 통일
