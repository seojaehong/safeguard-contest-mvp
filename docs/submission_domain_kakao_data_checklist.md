# SafeGuard 제출 직전 도메인·카카오·가점 데이터 체크리스트

## 1. safeclaw.kr 연결 작업

`safeclaw.kr`은 SafeGuard 제출용 대표 도메인으로 연결한다.

### Vercel에서 할 일

1. Vercel Dashboard에서 `safeguard-contest-mvp` 프로젝트를 연다.
2. `Settings -> Domains`로 이동한다.
3. `safeclaw.kr`을 추가한다.
4. Vercel이 안내하는 DNS 값을 확인한다.
5. 도메인 등록기관 DNS에 아래 중 Vercel 안내값을 넣는다.

일반 루트 도메인 연결:

```text
Type: A
Name: @
Value: 76.76.21.21
```

www 서브도메인 연결:

```text
Type: CNAME
Name: www
Value: cname.vercel-dns.com
```

### 확인 기준

- `https://safeclaw.kr` 접속 시 Vercel Production이 열린다.
- `https://www.safeclaw.kr`도 같은 서비스로 redirect 또는 정상 접속된다.
- Vercel Domains 화면에서 SSL 상태가 `Valid`가 된다.
- 제출서류 홈페이지 주소는 `https://safeclaw.kr`로 쓴다.

현재 로컬 CLI는 Vercel 로그인 토큰이 없어 도메인 자동 연결을 실행하지 못했다. Dashboard에서 연결하거나 `vercel login` 후 다시 CLI로 연결한다.

## 2. 카카오 연결 판단

사용자가 공유한 Kakao Developers `카카오톡 메시지 REST API`는 카카오톡 메시지 API가 맞다. 다만 SafeGuard의 현장 전파 주 채널로 쓰기에는 제약이 크다.

### Kakao Developers 메시지 API의 위치

- `나에게 보내기`: 로그인한 사용자 본인 카카오톡으로만 보낼 수 있다.
- `친구에게 보내기`: 카카오 소셜 권한, 친구 목록/피커, 사용자 동의가 필요하다.
- 친구에게 보내기는 같은 서비스 사용자 관계와 권한 심사 제약이 있고, 한 번에 보낼 수 있는 대상도 제한된다.

따라서 이 API는 SafeGuard에서 아래 용도에 적합하다.

- 관리자 본인에게 테스트 메시지 보내기
- 웹에서 카카오 공유 버튼 제공
- 앱 사용자가 직접 자신의 카카오 친구에게 공유하는 보조 기능

### SafeGuard 운영용 권장 경로

현장 작업자에게 휴대폰 번호 기반으로 안전공지를 보내는 목적이면 `알림톡` 또는 `친구톡` 공급사 경로가 맞다. 현재 구조에서는 Solapi를 쓰는 편이 가장 자연스럽다.

필요 값:

```text
SOLAPI_API_KEY=
SOLAPI_API_SECRET=
SOLAPI_SENDER=
SOLAPI_KAKAO_PFID=              # 또는 Solapi 콘솔의 카카오 채널 식별자
SOLAPI_KAKAO_TEMPLATE_ID=       # 승인된 알림톡 템플릿 ID
```

n8n에서 할 일:

1. `kakao` 채널 분기를 추가한다.
2. Solapi 카카오 알림톡 발송 API를 호출한다.
3. 승인 템플릿 변수에 SafeGuard 값을 매핑한다.
4. 실패 시 SMS 대체발송 또는 `kakao failed / sms fallback`으로 로그를 남긴다.

권장 템플릿 초안:

```text
[SafeGuard 현장 안전공지]
현장: #{siteName}
작업: #{workSummary}
핵심위험: #{topRisk}
필수조치: #{actions}
확인: TBM 및 당일 안전교육 내용 확인 후 작업을 시작하세요.
```

외국어 알림톡은 템플릿 승인 부담이 커질 수 있으므로, 제출 전에는 한국어 알림톡 1종을 먼저 승인하고 외국어는 SMS/메일/문서팩 링크로 운영하는 것이 안전하다.

## 3. 제출서류 AI 활용란

AI 활용 여부는 `사용`으로 체크한다.

활용 AI명:

- `제미나이`
- 필요 시 `기타`에 `OpenAI API fallback` 기재

작성 문장:

```text
SafeGuard는 Gemini API를 활용해 현장 작업 설명, 공공데이터 기반 근거, 기상 신호, 안전보건 자료를 종합하여 위험성평가표, TBM, 안전보건교육 기록, 외국인 근로자 안내문을 자동 생성한다. OpenAI API는 일부 요약·재생성 경로의 보조 fallback으로 둔다.
```

## 4. 국가중점 데이터 활용 판단

### 1순위: 법제처 법령정보 데이터

사용 권장: `가. 국가중점 데이터 활용`에 체크한다.

이유:

- 법제처 법령정보데이터는 현행법령, 행정규칙, 자치법규, 판례, 법령해석례, 행정심판례, 헌재결정례 등을 포함한다.
- SafeGuard는 `Law.go` 경로로 산업안전보건법, 시행령, 시행규칙, 판례, 법령해석례를 검색하고 문서팩의 근거로 연결한다.
- 제품 핵심인 위험성평가, TBM, 안전교육 기록의 법령 근거에 직접 반영된다.

신청서 입력 예시:

```text
데이터 보유 기관 분류: 고용부 및 산하기관이 아니라 국가행정기관 또는 기타 공공기관 분류 중 제공 양식에 맞춰 선택
기관명: 법제처
데이터 성격: 국가중점데이터
데이터명: 법령정보데이터 또는 국가법령정보
전체 항목명: 현행법령, 행정규칙, 판례, 법령해석례, 별표/서식, 법령용어, 법령연계정보
활용 내용: 산업안전보건법령·판례·해석례를 현장 작업 조건과 매칭하여 위험성평가표, TBM, 안전보건교육 기록의 반영 근거로 제공
```

주의:

- 공모전 입력 폼에서 `고용부 및 산하기관`만 선택 가능하면, 법제처는 별도 데이터 2 또는 기타 기관 데이터로 분리한다.
- 고용노동 공공데이터 필수 요건은 Work24, KOSHA, 고용노동부 법령/교육 데이터로 충족하고, 법제처 국가법령정보는 가점 데이터로 설명한다.

### 2순위: 한국에너지기술연구원 실시간 홍반자외선 데이터

사용 여부: 이번 제출에서는 `확장 후보`로 두는 것이 좋다.

장점:

- 국가중점 데이터로 표시되어 있어 가점 후보가 된다.
- 옥외 도장, 지붕, 도로, 조경, 시설관리 작업에서 폭염·자외선 노출 안전조치와 연결 가능하다.

단점:

- 현재 SafeGuard 핵심 플로우는 기상청 단기예보, 초단기실황, 초단기예보, 기상특보, 영향예보 중심이다.
- 홍반자외선은 모든 시나리오에 필요한 정보가 아니며, 급하게 붙이면 “가점용으로 억지 추가”처럼 보일 수 있다.

권장:

- 이번 제출 신청서의 국가중점 데이터는 법제처 법령정보를 1순위로 쓴다.
- 홍반자외선은 `옥외 작업 확장 데이터`로 문서에 언급한다.
- 기능 추가는 제출 후 `옥외작업 위험 신호` 카드에 붙인다.

### 기상청 생활기상지수 조회서비스

기상청 생활기상지수는 제품 적합성이 높다.

- 자외선지수
- 대기정체지수
- 여름철 체감온도

다만 국가중점 표시가 명확한 홍반자외선 데이터와 달리, 제출 가점용 주 데이터로 쓰기보다는 기상 위험 보강 데이터로 두는 편이 안전하다.
