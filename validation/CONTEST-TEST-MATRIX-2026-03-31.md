# SafeGuard 공모전 테스트 매트릭스 (2026-03-31)

## 목적
- SafeGuard를 "돌아가는 데모"가 아니라 **수상권 제출물/시연물** 기준으로 점검한다.
- 기능 성공 여부만이 아니라 **법적 정합성 / 근거 제시력 / 실패 시 방어력**까지 함께 평가한다.

## 평가 방식
- 각 항목은 `0/1/2점`으로 기록한다.
  - `0`: 실패 / 없음 / 위험
  - `1`: 부분 충족 / 수동 보완 필요
  - `2`: 안정적 충족 / 시연 가능
- 핵심 항목 합계 기준:
  - `32점 이상`: 수상권 시연 가능 수준
  - `24~31점`: MVP 강함, 보완 필요
  - `23점 이하`: 시연/심사에서 감점 위험 큼

## 평가 축
1. 기능 완주
2. 공공데이터 연결성
3. 법적 정합성
4. 근거/출처 표현
5. 실패 시 방어력
6. 시연 설명력

---

## 1. 골든패스 테스트
### GP-01 서울 옥외 철골 + 강풍 + 추락/지게차 + 벤젠
- 입력: 대표 시연 시나리오
- 기대 결과:
  - 위험성평가 초안 생성
  - TBM 브리핑 생성
  - TBM 일지 초안 생성
  - 기상 + KOSHA + 법령 근거 모두 연결
- 채점:
  - 0: 산출물 미완주
  - 1: 산출물은 나오나 일부 근거/데이터 축 누락
  - 2: 산출물/근거/설명 흐름 모두 시연 가능

### GP-02 해고/근로기준법 법률 검색 흐름
- 입력: `근로기준법 해고`
- 기대 결과:
  - 판례/해석례/법령 중 2개 축 이상 결과 반환
  - 출처 구분 표시
- 채점:
  - 0: 결과 빈약 또는 오류
  - 1: 결과는 있으나 출처/상세 연결 약함
  - 2: 결과 + 출처 + 상세 라우팅 모두 자연스러움

## 2. 법적 정합성 테스트
### LEG-01 과잉 단정 방지
- 기대 결과:
  - "위법이다" 같은 단정 대신
  - "관련 법령상 조치 검토 필요", "근거 확인 필요" 같은 실무형 표현 사용
- 채점:
  - 0: 단정 표현 다수
  - 1: 일부 잔존
  - 2: 전반적으로 안전한 표현 유지

### LEG-02 근거-산출물 연결
- 기대 결과:
  - 체크리스트/브리핑 문구가 관련 법령/가이드와 연결됨
- 채점:
  - 0: 근거 없는 산출물
  - 1: 일부만 연결
  - 2: 주요 문구가 근거와 연결됨

### LEG-03 법령/판례/해석례 source mix
- 기대 결과:
  - source label 또는 source mix가 확인 가능
  - 판례와 해석례가 구분됨
- 채점:
  - 0: 출처 불명
  - 1: 일부만 표기
  - 2: 명확히 구분

### LEG-04 산안법/중처법/가이드 확장 가능성
- 기대 결과:
  - 현재 구조가 산안법, 시행령, 시행규칙, 중처법, KOSHA 가이드를 수용 가능한 형태
- 채점:
  - 0: 구조 미흡
  - 1: 개념 수준
  - 2: 확장 포인트가 코드/문서에 반영됨
