# SafeGuard 방어용 기능 업그레이드 로드맵 (2026-03-31)

## 목표
공모전 심사/시연에서 흔히 맞는 질문과 실패 상황에 대비해, SafeGuard를 **설명 가능한 제품**으로 업그레이드한다.

## 원칙
- 기능 추가보다 **감점 방지**를 우선한다.
- 법적 판단을 단정하지 않는다.
- 근거/출처를 보여줄 수 있어야 한다.
- 외부 API 실패가 전체 붕괴로 이어지지 않게 한다.

---

## P0. 바로 해야 하는 것
### 1. source badge / source mix 고정
- 검색/답변/상세 화면에서
  - 법령
  - 판례
  - 해석례
  - KOSHA
  - 기상청
  출처를 분리 표시
- 목적:
  - "어디서 나온 말인지" 즉시 설명 가능

### 2. 법적 표현 guardrail
- 금지 표현 예시:
  - 위법이다
  - 반드시 불법이다
  - 처벌된다(단정)
- 권장 표현 예시:
  - 관련 법령상 검토 필요
  - 조치 필요 가능성 높음
  - 추가 확인 필요
- 목적:
  - 법적 단정에 따른 감점 방지

### 3. fallback 상태 표시
- 외부 API 실패 시
  - "일부 외부 근거는 현재 불안정하여 저장된/기본 근거 기준으로 표시 중"
  같은 안내 추가
- 목적:
  - 데모 실패를 제품의 정직한 방어로 전환

### 4. 골든패스 고정
- 대표 시연 시나리오를 1개 확정하고
- 해당 시나리오에서 항상 가장 잘 보이는 결과/근거 흐름을 보장
- 목적:
  - 발표장에서 흔들리지 않는 시연 확보

## P1. 바로 다음 단계
### 5. citation ranker
- 답변 생성 시
  - 법령
  - 판례
  - 해석례
  - KOSHA
  우선순위를 조절
- 목적:
  - 질문 유형별로 더 맞는 근거를 앞에 배치

### 6. 질문 정규화
- 예:
  - `근기법 해고` -> `근로기준법 해고`
  - `산안법 위험성평가` -> `산업안전보건법 위험성평가`
- 목적:
  - live hit-rate 개선

### 7. 상세 근거 펼침
- 검색 결과 카드에서
  - 핵심 요약
  - 상세 보기
  - 관련 조문/해석/판례 이동
- 목적:
  - "왜 이 결과가 나왔는지" 설명력 강화

### 8. 장애 로그/실패 로그 분리
- 429 / timeout / empty result / parse fail을 구분 기록
- 목적:
  - 심사 직전 안정화 포인트를 정확히 집기 위함

---

## P2. 있으면 강해지는 것
### 9. 근거 snapshot 저장
- 시연용 대표 질의에 대해
  - 법령/판례/해석례 핵심 응답을 snapshot처럼 보존
- 목적:
  - 발표 직전 외부 변동 리스크 완화

### 10. 역할별 브리핑 분화
- 관리자
- 반장
- 근로자
- 외국인 근로자
- 목적:
  - 실용성/사회적 가치 점수 강화

### 11. 산안법/중처법 requirement map 강화
- 산안법, 시행령, 시행규칙, 중처법, KOSHA 가이드를
  requirement -> checklist -> evidence 구조로 정리
- 목적:
  - SafetyClaw와 SafeGuard를 공통 법적 코퍼스로 연결

1. source badge / source mix 고정
2. 법적 표현 guardrail
3. fallback 상태 표시
4. 골든패스 점검
5. 질문 정규화
6. citation ranker
7. 상세 근거 펼침
8. snapshot 저장

## 성공 기준
- 심사위원이 "이건 어디 근거냐"라고 물었을 때 즉시 답 가능
- 외부 API 하나가 흔들려도 시연은 완주 가능
- AI가 임의로 판단하는 제품처럼 보이지 않음
- 법적 정합성과 실무성이 동시에 보임
