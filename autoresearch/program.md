# SafetyGuard Autoresearch Program v0

## Goal
Evolve the SafetyGuard contest/demo track into a clearer, more convincing onboarding surface.

## Current focus
- sharpen first impression and product story
- remove confusing dead ends
- improve demo flow and onboarding clarity

## Rules
1. Optimize for understandable demo value.
2. Prefer visible UX improvements over hidden complexity.
3. Keep changes reviewable.
4. Record each experiment and verdict.

## Success criteria
- improved first-run clarity
- cleaner demo flow
- build or local validation still passes where available
- experiment note recorded in autoresearch/runs

## First experiments
1. landing message clarity
2. answer panel/onboarding friction audit
3. demo path simplification
