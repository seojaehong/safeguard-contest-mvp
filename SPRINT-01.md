# SPRINT-01 — API 활용 대회 준비

## 목표
- `contest-mvp`를 제출용 1차 데모 기준으로 정리한다.

## 이번 스프린트 산출물
- 제품 포지셔닝 1개 확정
- 데모 시나리오 1개 확정
- API 연결 우선순위 확정
- mock/live fallback 정책 확정

## 현재 판단
- 가장 좋은 스토리는 `날씨 → 법령 → 현장조치` 산업안전 코파일럿이다.
- Law.go 단독보다, 기상청/KOSHA 축을 설명 가능한 구조가 대회용 메시지에 더 강하다.

## 구현 후보 우선순위
1. 질문형 결과에 `기상 위험 / 관련 법령 / 현장 조치` 구조 반영
2. live Law.go 검색 adapter 연결
3. 기상청 신호를 설명형 카드로 넣기
4. KOSHA 사례/가이드 fallback 추가

## 테스트 기준
- `/search`가 mock/live 둘 다 깨지지 않아야 함
- `/ask`에서 최소 1개 데모 질문에 대해 근거형 응답이 나와야 함
- live API 실패 시 mock 또는 설명형 fallback으로 내려가야 함
