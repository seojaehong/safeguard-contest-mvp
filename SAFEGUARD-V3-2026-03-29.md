# [제5회 고용노동 공공데이터·AI 활용 공모전 기획서 v3] 세이프가드(SafeGuard)

**부제: 중소규모 현장용 AI 위험성평가·TBM 코파일럿**  
**확장 포인트: 외국인 근로자를 위한 다국어 안전 브리핑 지원**

---

## 1. 제안 배경 및 필요성

### 1.1. 중소규모 현장의 산업안전 관리 공백
중소규모 건설·제조 현장은 안전관리 인력과 시간의 제약으로 인해, 매일 달라지는 작업 조건과 기상 변화에 맞춰 위험요인을 빠르게 재평가하기 어렵습니다. 특히 작업 시작 전 위험성평가와 TBM(Tool Box Meeting)은 형식적으로 반복되는 경우가 많고, 실제 위험요인과 연결되지 못하는 문제가 있습니다.

### 1.2. 외국인 근로자와 신규 작업자의 이해 격차
위험한 작업이 많은 현장에서는 외국인 근로자, 신규 작업자, 협력업체 인력이 함께 일하는 경우가 많습니다. 이때 안전 수칙이 현장 언어로 충분히 전달되지 않거나, 전달되더라도 실제 행동 지침으로 이어지지 못하는 문제가 발생합니다. 결국 안전 브리핑은 했지만 실질적 이해와 행동 변화로 이어지지 않는 경우가 생깁니다.

### 1.3. 현장 관리자의 의사결정 부담
현장 관리자는 기상특보, 법령, KOSHA 가이드, 과거 재해사례를 모두 따로 확인한 뒤 오늘 작업의 지속 여부와 주의사항을 판단해야 합니다. 그러나 현실적으로는 시간이 부족하고, 법적 근거와 현장 조치를 연결하는 데 부담이 큽니다.

### 1.4. 기존 솔루션의 한계
기존 안전 솔루션은 CCTV, 센서, 알림 중심이 많아 구축비용이 높고, 현장 관리자가 가장 자주 하는 업무인 **위험성평가·TBM 준비·브리핑·기록**까지 직접 도와주는 경우는 드뭅니다. 또한 기상·법령·KOSHA 데이터를 함께 엮어 오늘 작업의 맥락에 맞는 산출물을 생성하는 서비스는 부족합니다.

## 2. 서비스 개요

**세이프가드(SafeGuard)**는 고용노동·산업안전 공공데이터와 AI를 활용해, **오늘 작업의 위험을 분석하고 위험성평가 초안, TBM 브리핑, TBM 일지 초안까지 생성하는 현장형 안전 코파일럿**입니다.

핵심 흐름은 다음과 같습니다.
1. 오늘 작업 정보 입력  
2. 기상·작업·위험 키워드 분석  
3. 관련 법령 및 KOSHA 근거 연결  
4. 현장 조치 제안  
5. 위험성평가 / TBM 브리핑 / TBM 일지 산출물 생성

이 서비스는 단순 조회형 검색기가 아니라, **현장에서 바로 읽고, 말하고, 기록할 수 있는 안전 산출물 생성 도구**를 지향합니다.

## 3. 타겟 고객 및 가치 제안

| 타겟 | 주요 문제 | 세이프가드의 가치 |
|---|---|---|
| 현장 관리자 / 반장 | 기상 악화, 위험요인 변화에 대한 즉시 판단 부담 | 법령·KOSHA 근거 기반 위험 요약과 조치 가이드 제공 |
| 안전관리자 | 위험성평가/TBM 준비와 교육 메시지 작성 부담 | 위험성평가 초안, TBM 브리핑, 일지 초안 자동 보조 |
| 사업주 | 중대재해 대응 및 안전관리 입증 부담 | 선제적 위험 관리와 기록 보조로 관리 체계 지원 |
| 외국인 근로자 / 신규 작업자 | 언어 장벽, 안전지시 이해 부족 | 쉬운 문장 기반 다국어 브리핑 및 향후 TTS 확장 지원 |

## 4. 공공데이터 활용 구체성

세이프가드는 단순히 여러 데이터를 병렬로 나열하지 않고, **각 데이터의 역할을 분리해 하나의 판단 흐름으로 연결**합니다.

| 공공데이터 | 예시 API/엔드포인트 | 세이프가드 내 역할 | 산출물 반영 방식 |
|---|---|---|---|
| 기상청 단기예보 | `getUltraSrtNcst`, `getVilageFcst` 계열 | 온도/기상 상태 파악 | 오늘 작업 환경 요약 |
| 기상청 기상특보 | `getWthrWrnList` | 강풍/호우/건조/폭염 등 특보 탐지 | 위험도 상승 근거, 작업중지·조정 검토 |
| 기상청 영향예보 | `getHWImpactValue`, `getCWImpactValue` | 폭염/한파 건강영향 신호 보강 | 폭염/한파 대응 문구 강화 |
| KOSHA 안전보건법령 스마트검색 | `smartSearch` | 관련 안전보건 법령 탐색 | 법령 근거 블록 생성 |
| KOSHA 국내재해사례 | `disaster_api02` 계열 | 유사 사고/재해 사례 보강 | TBM 경고 포인트, 사고사례 인용 |
| KOSHA 사고사망 사례 | `news_api02` 계열 | 중대 사고 사례 요약 | 위험 경각심 및 교육 포인트 |
| KOSHA 물질안전보건자료 | `getChemList`, `getChemDetailXX` | 취급 물질의 규제·유해성 정보 파악 | MSDS 주의사항, 화학물질 위험 반영 |
| KOSHA 안전보건자료 링크 | `selectMediaList01` 계열 | 관련 가이드/홍보자료 참조 | 실무 가이드 근거 제시 |
| Law.go / 국가법령정보 | 관련 법령/판례 조회 | 법률/규정 근거 보강 | 조문 인용 및 판단 근거 설명 |

핵심은 **기상청=위험 신호**, **KOSHA=실무 안전 데이터**, **법령 API=근거 정리**의 역할 분담입니다.

## 5. 핵심 기능

### 5.1. 통합 위험 신호 수집
- 기상청 API: 단기예보, 기상특보, 영향예보
- KOSHA API: 안전보건법령 스마트검색, 국내재해사례, 사고사망 사례, 안전보건자료 링크, MSDS
- 법령 데이터: Law.go / 국가법령정보

### 5.2. 날씨-법령-KOSHA 매핑 엔진
세이프가드는 단순한 날씨 알림이 아니라, 위험 신호를 법령과 실무 가이드에 연결합니다.
- 폭염/한파: 산업안전보건법 제39조 등 직접 근거 중심 안내
- 강풍/호우/낙뢰: 작업중지·피난 및 KOSHA 가이드 기반 조치 검토 안내
- 화학물질 취급 시: MSDS 규제 정보 및 취급상 주의사항 보강

중요한 점은 법 위반을 단정하는 것이 아니라, **관련 법령상 조치 검토 필요**라는 실무형 표현으로 판단을 지원하는 것입니다.

### 5.3. 위험성평가 초안 생성
- 작업명
- 유해위험요인
- 위험도
- 감소대책
- 관련 법령/가이드 근거

### 5.4. TBM 브리핑 생성
- 오늘 핵심 위험 3개
- 반드시 지킬 조치
- 날씨 관련 추가 주의사항
- 현장 반장/관리감독자용 읽기 문안

### 5.5. TBM 일지 초안 생성
- 일시
- 작업내용
- 참석자
- 전달내용
- 확인사항
- 특이사항
- 후속조치

### 5.6. 실질적 TBM 유도
예시 질문:
- 오늘 작업에서 가장 큰 위험은 무엇인가?
- 강풍 시 중지 검토 기준은 무엇인가?
- 신규 작업자에게 다시 설명해야 할 핵심은 무엇인가?

### 5.7. 다국어 브리핑 확장
- 쉬운 문장 기반 다국어 안내
- 향후 TTS(Text-to-Speech) 음성 브리핑으로 확장
- 화면 비의존 환경을 고려한 전달 채널 확장

## 6. 대표 시연 시나리오

### 시나리오명
**서울 도심 옥외 작업 + 강풍주의보 + 추락/지게차 위험**

### 입력
- 위치: 서울
- 작업 유형: 옥외 작업
- 작업 시간: 주간
- 위험 키워드: 추락, 지게차
- 선택 정보: 벤젠 취급
- 기상 조건: 강풍주의보

### 시스템 흐름
1. 사용자가 오늘 작업 정보를 입력한다.  
2. 시스템은 기상특보와 작업 위험 키워드를 읽는다.  
3. 관련 법령, KOSHA 가이드, 재해사례를 연결한다.  
4. 오늘 핵심 위험 3개와 즉시 조치를 요약한다.  
5. 위험성평가 초안을 생성한다.  
6. TBM 브리핑 문안을 생성한다.  
7. TBM 일지 초안을 생성한다.  
8. 필요 시 외국인 근로자용 쉬운 문장 브리핑으로 변환한다.

### 시연에서 보여줄 산출물
- 위험성평가 초안
- TBM 브리핑 문안
- TBM 일지 초안
- (확장) 외국인 근로자용 다국어 안내

이 시나리오는 공공데이터 활용, 실용성, 산출물 생성, 사회적 가치가 모두 드러나는 골든패스입니다.

## 8. 구현 현황 및 MVP 완성도

세이프가드는 아이디어 수준을 넘어, **실제 공공데이터 API 연동 및 E2E 프로토타입 검증을 완료한 상태**입니다.

### 현재 구현 완료 항목
- `contest-mvp` 웹앱 스캐폴드 구현
- `/search`, `/ask`, `/law/[id]`, `/precedent/[id]` 라우트 구성
- mock 기반 질문/검색 흐름 구성
- 별도 E2E 프로토타입 구현
  - `mvp_e2e.py`
  - `sample_input.json`
  - `weather_legal_mapping.json`
- 실 API 테스트 결과 파일 확보
  - `e2e_output.json`
  - `dry_run_output.json`

### 검증 결과
- 기상청/KOSHA/법령 API 일부는 실제 라이브 호출 성공
- 일부 API는 `429` 또는 timeout 발생
- 그러나 dry-run 시나리오는 끝까지 완주 가능
- 즉, **시연 가능한 흐름은 이미 확보되었고, 안정화는 다음 단계**입니다.

### 현재 수준 표현
본 프로젝트는 “즉시 상용화 완료”보다는, **실 API 기반 E2E 프로토타입과 시연 가능한 MVP 골격을 확보한 상태**라고 보는 것이 정확합니다.

## 9. 기대효과 (정량 목표 포함)

본 서비스는 중소규모 현장의 반복적 안전 문서 업무를 줄이고, 브리핑의 실질성을 높이는 것을 목표로 합니다.

### 정량 목표(시연/PoC 기준 가정)
- 위험성평가 초안 작성 시간: **30분 → 5~10분 수준으로 단축**
- TBM 브리핑 문안 준비 시간: **10분 → 1분 내 생성**
- 안전 담당자의 수작업 정보 수집 단계: **기상/법령/가이드 개별 조회 → 단일 흐름 통합**
- 외국인 근로자용 안전 안내 작성: **별도 수작업 번역 필요 → 쉬운 문장 기반 자동 초안 제공**

### 기대효과
- 산업안전 담당자의 판단 속도 향상
- 법령 근거 기반 브리핑 자동화
- TBM의 실질성 강화
- 위험성평가 초안 작성 부담 감소
- 교육/점검/감사 대응 기록 품질 향상

※ 위 수치는 실제 현장 PoC를 통해 향후 보정될 예정이며, 현재는 시연 기준 목표치입니다.

## 11. 향후 고도화 로드맵

### Phase 1. 공모전 제출 시점
- 대표 데모 시나리오 1개 고정
- 위험성평가/TBM 브리핑/TBM 일지 산출물 시연
- retry/backoff 및 fallback 정책 보강
- 시연용 웹 프로토타입 고도화

### Phase 2. 단기 고도화
- 외국인 근로자 대상 다국어 브리핑 강화
- TTS 연동

## 12. 결론

세이프가드는 공공데이터를 단순히 검색·조회하는 서비스가 아니라, **오늘 현장의 위험을 읽고, 관련 근거를 제시하며, 위험성평가와 TBM 산출물로 연결하는 실무형 안전 코파일럿**입니다.

여기에 외국인 근로자를 위한 다국어 브리핑 기능을 더함으로써, 사회적 가치와 현장 적용성을 동시에 강화할 수 있습니다.

즉, 세이프가드는 **고용노동 공공데이터·AI 활용 공모전의 취지인 사회문제 해결, 실용성, AI 활용성, 공공데이터 활용성**을 모두 충족할 수 있는 현실적인 서비스 제안입니다.
