# contest-mvp v2 로드맵

## v2 목표
`contest-mvp`를 **브리핑 가능한 MVP**에서 **심사위원에게 시연 가능한 데모 MVP**로 끌어올린다.

핵심은 아래 4가지를 한 흐름으로 묶는 것이다.
1. 위험 신호 입력
2. 법령/KOSHA 근거 연결
3. 현장 조치 제안
4. 위험성평가/TBM 산출물 생성

## v2 제품 정의
- 제품명: **산업안전 위험성평가·TBM 코파일럿**
- 타겟: 중소규모 건설/제조 현장, 안전관리자/관리감독자/현장소장
- 포지션: 고용노동·산업안전 공공데이터와 AI를 활용해 오늘 작업의 위험을 실무 산출물로 번역하는 현장형 도우미

## v2 데모 기준
심사위원 앞에서 아래 흐름이 보여야 한다.

1. 사용자가 오늘 작업 정보를 입력한다.
2. 시스템이 기상/작업/위험 키워드를 읽는다.
3. 관련 법령과 KOSHA 근거를 제시한다.
4. 오늘의 핵심 위험 3개를 요약한다.
5. 위험성평가 초안을 생성한다.
6. TBM 브리핑 문안을 생성한다.
7. TBM 일지 초안을 생성한다.
